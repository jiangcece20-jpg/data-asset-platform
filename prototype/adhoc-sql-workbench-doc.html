一级菜单能力 支持多表 SQL / JOIN 在线查看 + 草稿 + 历史

即席查询工作台

这是一份为“数据资产管理平台”准备的交互式需求文档。目标是把现有的单表 SQL 查询页升级为一个全局查询工作台， 新增顶部一级菜单，承接用户对已有权限数据表的多表查询分析需求。

菜单位置
放在“我的”前面，作为一级菜单
查询范围
仅支持查询用户已有权限的表
能力边界
支持多表 SQL 与 JOIN，不做导出和分享
沉淀能力
支持保存 SQL 草稿和查看历史查询
文档导航

核心定位

将现有 SQL 页面升级为全局查询工作台，而不是重新建设一套独立查询产品。

用户价值

把“找表、看详情、申请权限、发起查询”串成完整链路，承接业务分析的高频查询需求。

实施策略

优先复用现有 `page-sql` 与 `page-sql-noperm`，再补齐全局入口、草稿、历史、权限校验和多表辅助。
