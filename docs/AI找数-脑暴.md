# AI 找数 — 脑暴速写

## 核心目标与动机

当前平台的"资源发现"依赖关键词搜索 + 筛选条件，用户需要知道准确的表名、英文名或分类才能找到数据。AI 找数要解决的是：**用户只知道业务问题，不知道该找哪张表**的场景。

## 要解决的问题

1. 用户描述的是业务语言（"我想看用户下单后的退款率"），但数据资产用的是技术语言（`dwd_order_refund_detail`）
2. 不同角色关心的资源类型不同 — 分析师找报表，开发找表和 API，运营找看板
3. 现有搜索是精确匹配思路，缺乏语义理解和推荐能力
4. 用户找到一个资源后，不知道还有哪些相关的上下游资源可以配合使用
5. 平台已有数千个资源，靠浏览和筛选效率低下

## 目标用户画像

| 角色 | 典型场景 | 期望推荐内容 |
|---|---|---|
| 数据分析师 | "帮我找用户留存相关的数据" | 报表 > 宽表 > 指标口径 |
| 数仓/数据开发 | "订单域有哪些 DWD 层的表" | 表 > 视图 > 血缘上下游 |
| 后端开发 | "有没有用户画像相关的 API" | API > 表 |
| 业务运营 | "我想看各渠道的 GMV 表现" | 报表 > 看板 |
| 产品经理 | "核心转化漏斗数据在哪" | 报表 > 指标口径 > 看板 |
| 管理层 | "公司核心经营数据" | 报表 > 看板 |

## 使用场景

1. 分析师点击"AI 找数"，输入"我想分析用户从注册到首单的转化漏斗"，AI 返回相关的用户行为表、订单表、以及已有的转化漏斗报表
2. 运营同学输入"各城市骑手配送时效"，AI 理解意图后推荐配送域的报表和看板，而不是底层 ODS 表
3. 开发同学输入"用户标签服务"，AI 优先推荐 API 接口，其次推荐底层标签表
4. 用户输入模糊需求"用户相关的数据"，AI 追问"你想看的是用户行为、用户画像、还是用户交易？"

## 关键决策记录

- 入口：嵌入现有搜索框旁，点击弹出中央对话弹窗
- 交互模式：重交互，支持多轮对话，就地闭环
- 角色识别：系统自动识别（部门+职位+业务线+行为数据），用户无感
- 角色粒度：六种细分角色（分析师、数仓开发、后端开发、运营、产品、管理层）
- 资源范围：表、视图、API、报表、看板、指标口径、标签
- 不做指标值查询（管理层直接找报表）
- 追问策略：置信度判断，有把握直接给结果，没把握才追问
- 卡片信息：名称、类型、匹配原因、来源系统（精简四要素）
- 卡片点击：弹窗内展开摘要，确认后再跳详情页
- 对话历史：持久保留
- 后端方案：LLM + 向量检索混合（本期原型先 mock）
