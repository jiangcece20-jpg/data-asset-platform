# 指标、标签、API 详情页需求变更说明

## 1. 变更目的

本说明用于明确相对于“统一表类详情页模型”的变更内容，帮助产品、设计、研发、测试在评审和落地时快速理解：

- 为什么要改
- 改了哪些
- 哪些保留
- 哪些删除
- 对页面结构、交互和血缘抽屉分别有什么影响

---

## 2. 变更背景

原有详情页以表类资产为主模型：

- 首个 Tab 默认是字段结构
- 血缘抽屉默认有字段信息页签
- 右侧固定栏强调数据层级、分区、DDL、存储等表类信息

该结构适合表，但不适合指标、标签、API：

- 指标不应优先看字段，而应优先看口径
- 标签不应优先看血缘，而应优先看规则与值域
- API 不应展示字段结构，而应展示参数与返回说明

因此本次改动本质上是从“统一表模型”升级为“统一框架 + 类型化内容”。

---

## 3. 变更范围总览

本次变更覆盖以下 4 个区域：

1. 详情页顶部 Tab 结构
2. 详情页首个 Tab 内容
3. 右侧固定信息栏
4. 血缘图谱节点抽屉页签与内容

同时补充：

- 标签类型的 mock 数据与入口

---

## 4. 详情页 Tab 变更

## 4.1 变更前

所有类型默认沿用表类 Tab：

- 字段结构
- 样例数据
- 分区信息
- 血缘关系
- 使用说明
- DDL变更
- 操作记录

## 4.2 变更后

### 表 / 视图

保持原结构：

- 字段结构
- 样例数据
- 分区信息
- 血缘关系
- 使用说明
- DDL变更
- 操作记录

### 指标

调整为：

- 指标定义
- 血缘关系
- 使用说明
- 操作记录

### 标签

调整为：

- 标签定义
- 血缘关系
- 使用说明
- 操作记录

### API

调整为：

- 参数信息
- 血缘关系
- 使用说明
- 操作记录

### 报表

同步调整为：

- 报表定义
- 血缘关系
- 使用说明
- 操作记录

## 4.3 变更原因

- 非表类对象不应继续强行复用表类结构
- 首个 Tab 应承接“解释对象”的职责
- 血缘保留，但不作为指标/标签默认首屏

---

## 5. 首个 Tab 内容变更

## 5.1 指标

### 删除 / 不再展示

- 默认字段结构首屏
- 样例数据
- 分区信息
- DDL变更

### 新增 / 调整为

- 指标定义头图区
- 口径概览
- 计算逻辑 / 指标公式
- 适用场景与分析维度
- 使用提醒

### 影响

- 用户更容易先理解指标口径
- 减少“先看血缘但不知道指标是什么”的理解障碍

## 5.2 标签

### 删除 / 不再展示

- 默认血缘首屏
- 表类字段结构模型
- 样例数据
- 分区信息
- DDL变更

### 新增 / 调整为

- 标签定义头图区
- 打标对象 / 标签类型 / 更新方式
- 标签值域
- 打标规则
- 使用建议与覆盖情况

### 影响

- 用户先理解标签规则，再看血缘，信息顺序更合理

## 5.3 API

### 删除 / 不再展示

- 字段结构命名
- 表类字段解释方式
- 样例数据
- 分区信息
- DDL变更

### 新增 / 调整为

- 参数信息头图区
- 接口概览
- 请求参数表
- 返回字段表
- 调用说明

### 影响

- 用户可以按 API 文档思路理解对象，不会把 API 当成表去看

---

## 6. 右侧固定信息栏变更

## 6.1 变更前

右侧固定栏偏表类字段，包括：

- 数据层级
- 是否分区
- 存储路径
- 生命周期
- 创建时间

这些信息对指标、标签、API 并不适用，甚至会造成误导。

## 6.2 变更后

### 指标

- 中文名
- 指标类型
- 统计粒度
- 统计周期
- 更新频率
- 来源平台
- 负责人
- 口径状态
- 近30天查询量
- 近30天浏览量

### 标签

- 中文名
- 打标对象
- 标签类型
- 更新方式
- 更新频率
- 来源平台
- 负责人
- 安全等级
- 近30天查询量
- 近30天浏览量

### API

- 中文名
- 描述
- 服务归属
- 请求方式
- 版本
- 鉴权方式
- 来源系统
- 负责人
- 近30天调用量
- 近30天浏览量

### 报表

- 中文名
- 描述
- 所属系统
- 刷新频率
- 来源平台
- 负责人
- 近30天查看量
- 近30天浏览量

## 6.3 变更原因

- 固定栏应展示跨 Tab 都会反复参考的信息
- 类型专属的核心信息应保留
- 表类专属字段不能继续扩散到所有类型

---

## 7. 血缘抽屉变更

## 7.1 变更前

血缘抽屉默认固定三页签：

- 基本信息
- 字段信息
- 血缘关系

这对 API、指标、标签并不合理。

## 7.2 变更后

### 表 / 视图

保留：

- 基本信息
- 字段信息
- 血缘关系

### API

改为：

- 基本信息
- 参数信息
- 血缘关系

### 指标 / 标签 / 报表

改为：

- 基本信息
- 血缘关系

## 7.3 具体变化

### 删除

- 指标节点的字段信息页签
- 标签节点的字段信息页签
- 报表节点的字段信息页签

### 新增

- API 节点参数信息页签

### 保留

- 所有类型都保留血缘关系页签
- 所有类型都保留基本信息页签

## 7.4 交互变化

- 抽屉打开时，根据当前节点类型动态生成页签
- 若上一个节点支持的页签在当前节点中不存在，则自动切换到第一个可用页签
- 内嵌血缘抽屉与全屏血缘抽屉保持一致

---

## 8. mock 与入口变更

### 新增 mock

- 新增标签资产：`label_user_value_tier`
- 新增标签资源：`r009`
- 新增标签血缘节点：`label_user_value_tier`

### 新增入口

- 资产目录类型筛选新增“标签”
- 资源发现类型筛选新增“标签”

### 目的

- 保证新结构不只是代码逻辑存在，同时在原型中可以直接被访问和查看

---

## 9. 页面视觉变化说明

本次除了结构变化，也同步做了定义型页面的视觉强化：

- 指标定义、标签定义、参数信息、报表定义增加头图区
- 用关键属性卡片强化“先看重点”的节奏
- 指标公式、API 参数表、标签规则区块做更明显的主块展示
- 首个 Tab 从“普通信息堆叠”提升为“定义页”视觉风格

目的不是做新风格，而是让用户在首屏更快抓到对象核心。

---

## 10. 对研发实现的影响

### 10.1 结构实现方式

建议使用配置驱动，而不是分散 `if else`：

- 详情页 Tab 配置函数
- 血缘抽屉页签配置函数
- 右侧固定栏配置函数

### 10.2 复用原则

- 类型判断逻辑应复用
- 详情页和血缘抽屉的类型配置应共享
- 内容模块可以复用通用卡片、通用表格、通用标签组件

### 10.3 测试重点

- 默认首个 Tab 是否正确
- 不同类型切换详情页时，右侧栏是否正确
- 血缘抽屉页签是否按节点类型切换
- API 节点是否展示参数信息而不是字段信息
- 指标/标签节点是否不再展示字段页签

---

## 11. 风险与注意事项

- 若后续增加更多资产类型，需要继续沿用类型化配置模式
- 指标、标签、API 当前为 mock 内容，后续真实平台接入时字段可能需要重新映射
- 若未来用户对 API 有更强文档需求，建议二期增加“错误码说明”“示例请求/响应”“调试入口”
- 若未来标签平台对标签规则支持更强，建议二期拆出“规则说明”独立 Tab

---

## 12. 总结

本次变更的核心收益是：

- 从“所有详情页都像表”升级为“不同类型有不同的主信息”
- 从“血缘优先”调整为“定义优先、血缘验证”
- 从“抽屉固定三页签”升级为“抽屉按节点类型切换”

这不仅是页面改版，更是详情信息架构从单一表模型向多资产类型模型的升级。
