# 资源与标签来源字段字典

## 1. 文档目标

本文档用于统一定义“资源来源”和“标签来源”的字段模型，供以下场景复用：

- 原型页面字段展示
- 产品 PRD
- 后端数据模型设计
- 前端筛选项与枚举维护

当前设计遵循“最小可用、可扩展”原则：

- 资源来源单独建模
- 标签来源单独建模
- 标签来源系统当前统一为 `画像标签系统`
- 标签来源类型当前仅保留 `系统生产` 和 `人工标注`

---

## 2. 资源主表字段字典

### 2.1 资源对象定义

资源与资产本质上是同一对象的不同状态：

- 资源：尚未上架到资产目录，或仍处于治理过程中的数据对象
- 资产：已上架成功的资源

因此资源主表建议承载统一对象信息，资产只是其状态之一。

### 2.2 字段定义

| 字段名称 | 字段编码 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|---|
| 资源ID | `resource_id` | string | 是 | 资源唯一标识 | `r001` |
| 资源英文名 | `resource_name` | string | 是 | 技术名称/物理名称 | `dwd_order_detail` |
| 资源中文名 | `resource_display_name` | string | 否 | 业务可读名称 | `订单明细宽表` |
| 资源类型 | `resource_type` | enum | 是 | 资源的对象类型 | `table` |
| 当前状态 | `resource_status` | enum | 是 | 当前治理状态 | `maintain` |
| 当前形态 | `resource_form` | enum | 是 | 资源当前是否已成为资产 | `resource` / `asset` |
| 来源类型 | `source_type` | enum | 是 | 资源来自哪一类系统 | `数仓引擎` |
| 来源系统 | `source_system` | string | 是 | 资源具体来自哪个系统/产品 | `MaxCompute` |
| 来源实例 | `source_instance` | string | 否 | 来源的具体实例、项目、库、空间 | `mc_prod_trade` |
| 来源路径 | `source_path` | string | 否 | 资源在来源系统中的具体定位 | `dw.trade.dwd_order_detail` |
| 技术负责人 | `tech_owner` | string | 否 | 资源维护责任人 | `李四` |
| 业务负责人 | `biz_owner` | string | 否 | 业务口径责任人 | `王五` |
| 目录归属 | `catalog_path` | string | 否 | 当前资源归属目录 | `交易域/订单/订单明细` |
| 更新时间 | `updated_at` | datetime | 否 | 最近一次更新时间 | `2026-03-27 10:00:00` |
| 简介 | `resource_summary` | string | 否 | 资源摘要说明 | `包含订单全生命周期信息` |

### 2.3 资源主表关键字段说明

#### `resource_type`

建议枚举：

- `db`
- `table`
- `view`
- `api`
- `report`
- `metric`

#### `resource_status`

建议枚举：

- `maintain`
  - 待维护
- `reviewing`
  - 待审核
- `rejected`
  - 已驳回
- `published`
  - 已上架
- `no_list`
  - 不上架
- `offline`
  - 已下架

#### `resource_form`

建议枚举：

- `resource`
  - 资源态，尚未成为资产
- `asset`
  - 资产态，已上架

#### `source_type`

建议枚举：

- `warehouse_engine`
  - 数仓引擎
- `analytic_db`
  - 分析型数据库
- `business_db`
  - 业务数据库
- `report_system`
  - 报表系统
- `api_service`
  - 接口服务
- `message_stream`
  - 消息/流系统
- `file_storage`
  - 文件/对象存储
- `metric_platform`
  - 指标平台

### 2.4 资源来源示例

| 资源类型 | 来源类型 | 来源系统 | 示例说明 |
|---|---|---|---|
| 表 | `warehouse_engine` | `MaxCompute` | 数仓明细表 |
| 表 | `analytic_db` | `SelectDB` | 实时分析表 |
| 表 | `business_db` | `MySQL` | 业务库原始表 |
| 视图 | `warehouse_engine` | `MaxCompute` | 数仓视图 |
| API | `api_service` | `内部微服务` | 用户信息接口 |
| 报表 | `report_system` | `万联灵析` | 经营分析报表 |
| 指标 | `metric_platform` | `指标平台` | 统一指标口径 |
| 指标 | `warehouse_engine` | `MaxCompute` | 数仓计算指标 |

## 3. 标签表字段字典

### 3.1 标签对象定义

标签不直接复用资源来源字段，单独定义来源体系。

原因：

- 同一个资源上的不同标签，来源可能不同
- 标签可能由系统自动生成，也可能由人工补充
- 后续还可能引入“系统生成后人工确认”的流程

### 3.2 字段定义

| 字段名称 | 字段编码 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|---|
| 标签ID | `tag_id` | string | 是 | 标签记录唯一标识 | `tag_001` |
| 资源ID | `resource_id` | string | 是 | 标签所属资源ID | `r001` |
| 标签名称 | `tag_name` | string | 是 | 标签文本值 | `交易域` |
| 标签分类 | `tag_category` | enum | 是 | 标签业务分类 | `business` |
| 标签来源类型 | `tag_source_type` | enum | 是 | 标签是系统生成还是人工打标 | `system_generated` |
| 标签来源系统 | `tag_source_system` | string | 是 | 标签来自哪个系统 | `画像标签系统` |
| 标签确认状态 | `tag_confirm_status` | enum | 否 | 标签是否经过人工确认 | `system_created` |
| 生效状态 | `tag_status` | enum | 否 | 标签是否生效 | `active` |
| 创建人 | `created_by` | string | 否 | 人工打标时记录操作人 | `张三` |
| 创建时间 | `created_at` | datetime | 否 | 标签生成时间 | `2026-03-27 10:00:00` |
| 更新时间 | `updated_at` | datetime | 否 | 标签更新时间 | `2026-03-27 11:00:00` |

### 3.3 标签字段关键说明

#### `tag_category`

建议枚举：

- `business`
  - 业务标签
- `technical`
  - 技术标签
- `governance`
  - 治理标签
- `profile`
  - 画像标签

#### `tag_source_type`

当前建议仅保留两类：

- `system_generated`
  - 系统生产
- `manual_labeled`
  - 人工标注

#### `tag_source_system`

当前固定值建议：

- `画像标签系统`

说明：

- 当前标签来源都来自画像标签系统
- 不论是系统自动生产，还是人工打标，来源系统都统一记录为 `画像标签系统`

#### `tag_confirm_status`

建议枚举：

- `system_created`
  - 系统已生成
- `manual_confirmed`
  - 人工已确认
- `manual_corrected`
  - 人工已修正

#### `tag_status`

建议枚举：

- `active`
  - 生效
- `inactive`
  - 失效

### 3.4 标签来源示例

| 标签名称 | 标签分类 | 标签来源类型 | 标签来源系统 | 标签确认状态 |
|---|---|---|---|---|
| 交易域 | `business` | `system_generated` | `画像标签系统` | `system_created` |
| 核心资源 | `governance` | `manual_labeled` | `画像标签系统` | `manual_confirmed` |
| 高价值候选 | `profile` | `system_generated` | `画像标签系统` | `manual_corrected` |

## 4. 枚举值字典表

## 4.1 资源类型枚举

| 枚举编码 | 枚举名称 | 说明 |
|---|---|---|
| `db` | 库 | 数据库/数据集 |
| `table` | 表 | 数据表 |
| `view` | 视图 | 数据视图 |
| `api` | API | 接口服务 |
| `report` | 报表 | 报表对象 |
| `metric` | 指标 | 指标对象 |

## 4.2 资源状态枚举

| 枚举编码 | 枚举名称 | 说明 |
|---|---|---|
| `maintain` | 待维护 | 资源已采集，待补充信息 |
| `reviewing` | 待审核 | 已提交上架申请，等待审核 |
| `rejected` | 已驳回 | 审核未通过，可重新提交 |
| `published` | 已上架 | 已进入资产目录 |
| `no_list` | 不上架 | 不纳入资产目录 |
| `offline` | 已下架 | 曾上架，当前已下架 |

## 4.3 资源形态枚举

| 枚举编码 | 枚举名称 | 说明 |
|---|---|---|
| `resource` | 资源 | 资源态 |
| `asset` | 资产 | 资产态 |

## 4.4 资源来源类型枚举

| 枚举编码 | 枚举名称 | 说明 |
|---|---|---|
| `warehouse_engine` | 数仓引擎 | 如 MaxCompute、Hive |
| `analytic_db` | 分析型数据库 | 如 SelectDB、StarRocks |
| `business_db` | 业务数据库 | 如 MySQL、Oracle |
| `report_system` | 报表系统 | 如 万联灵析 |
| `api_service` | 接口服务 | 内部服务/API 网关 |
| `message_stream` | 消息/流系统 | Kafka 等 |
| `file_storage` | 文件/对象存储 | OSS、S3、HDFS |
| `metric_platform` | 指标平台 | 统一指标平台 |

## 4.5 标签分类枚举

| 枚举编码 | 枚举名称 | 说明 |
|---|---|---|
| `business` | 业务标签 | 业务域、场景、主题 |
| `technical` | 技术标签 | 引擎、平台、技术属性 |
| `governance` | 治理标签 | 核心、临时、质量类 |
| `profile` | 画像标签 | 来源于画像体系的标签 |

## 4.6 标签来源类型枚举

| 枚举编码 | 枚举名称 | 说明 |
|---|---|---|
| `system_generated` | 系统生产 | 平台自动生成 |
| `manual_labeled` | 人工标注 | 人工新增或人工维护 |

## 4.7 标签确认状态枚举

| 枚举编码 | 枚举名称 | 说明 |
|---|---|---|
| `system_created` | 系统已生成 | 自动生成后未人工确认 |
| `manual_confirmed` | 人工已确认 | 自动标签经人工确认 |
| `manual_corrected` | 人工已修正 | 自动标签经人工修正或人工重打 |

## 5. 建议的原型展示方式

### 5.1 在资源列表中展示

建议展示：

- 来源类型
- 来源系统

例如：

- `数仓引擎 / MaxCompute`
- `分析型数据库 / SelectDB`
- `报表系统 / 万联灵析`

### 5.2 在详情页中展示

资源详情页建议新增区块：

- 来源信息
  - 来源类型
  - 来源系统
  - 来源实例
  - 来源路径

标签信息建议展示：

- 标签名称
- 标签分类
- 标签来源类型
- 标签来源系统
- 标签确认状态

---

## 6. 当前建议的最小落地结论

- 资源来源和标签来源必须分开建模
- 资源来源字段最少包含：
  - `source_type`
  - `source_system`
- 标签来源字段最少包含：
  - `tag_source_type`
  - `tag_source_system`
- 标签来源系统当前统一为：
  - `画像标签系统`
- 标签来源类型当前统一只保留：
  - `system_generated`
  - `manual_labeled`
