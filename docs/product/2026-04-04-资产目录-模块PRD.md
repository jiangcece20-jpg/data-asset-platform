## 2. 模块定位与背景

### 2.1 一句话定位

**资产目录**是面向业务与消费侧的**已上架资产浏览入口**：用户沿左侧**业务线目录树**定位到域/主题，在右侧通过关键词、类型、负责人、排序等缩小范围，在卡片列表中理解资产摘要并进入**统一详情页**。

### 2.2 要解决的问题

- 用户需要按**业务域、产品线、主题**组织方式找数，而不是按集群/库等物理路径（后者在资源发现）。
- 需要与「资源发现」明确分工：本模块**仅呈现资产**（`domain=asset`），不混入未资产化资源列表。
- 在列表中快速看到**目录归属、热度、负责人、类型**，减少无效点开详情。

### 2.3 非目标（本模块不做）

- 资源侧对象展示、物理结构树、范围「仅资源」（均在**资源发现**）。
- 上架、驳回、批量改目录、目录树 CRUD（均在**资源管理**及相关审批流）。
- 标签后台维护、标签字典管理（另模块/后台）。
- 原型中「标签筛选」若为全局 `showNotification` 占位，**一期可保留入口**但须在实现阶段替换为真实筛选或明确二期排期（与资源发现策略一致）。

## 3. 目标与成功标准

### 3.1 产品目标

- 用户能在**左侧树 + 右侧筛选**组合下，**几步内**打开目标资产详情。
- 列表与树节点上的**数量提示**（若有）在逻辑上自洽：同一筛选条件下，列表总条数与产品定义一致。
- 卡片信息与详情页表头**字段口径一致**（名称、目录、负责人、类型等）。

### 3.2 成功指标（可后续量化）

- 从资产目录进入详情的占比、树节点点击率、各筛选维度使用率。
- 「我的表」使用率（反映负责人视角效率）。

## 4. 用户与场景

### 4.1 主要用户

| 角色 | 典型诉求 |
|------|----------|
| 业务分析师 | 按业务线目录找到域内核心表、指标、报表 |
| 数据开发 | 在熟悉业务分类下浏览已上架资产、复制技术名 |
| 资产负责人 | 用「我的表」快速过滤自己负责范围 |

### 4.2 典型场景（用户故事摘要）

1. 展开业务线目录，点选某子域，列表仅展示该目录（及子路径）下的资产。
2. 在「全部」根下用关键词 + 类型 Tab 找某张表。
3. 从卡片点「申请权限」打开权限弹窗/流程（与详情页动作一致，见总册）。
4. 点击「管理」进入有权限用户的目录治理界面（原型为提示文案，实现可为路由跳转）。

## 5. 信息架构与页面结构

### 5.1 整体布局

- **左侧栏**（`sidebar-panel`）：标题「业务线目录」、**管理**按钮、`#asset-tree` 树形容器。
- **右侧主区**（`catalog-content`）：
  - **面包屑**（`#asset-breadcrumb`）：默认「全部」可点击重置上下文。
  - **筛选条**（`filter-bar`）：关键词、负责人下拉、**我的表**、排序（顺序按钮 + 字段下拉）、标签筛选入口。
  - **类型 Tab**（`#asset-type-tabs`）：全部、表、视图、API、报表、指标、标签。
  - **卡片列表**（`#asset-card-list`）+ **分页**（`#asset-pagination`）。

### 5.2 与相邻模块的关系

| 模块 | 关系 |
|------|------|
| 资源发现 | 可浏览资产+资源及物理树；本模块**仅资产 + 业务目录树**。 |
| 资产检索 | 关键词直达；本模块强调**树状业务导航 + 列表细筛**。 |
| 统一详情页 | 卡片点击 `openDetail('asset', id)`；表头与卡片字段对齐。 |
| 资源管理 | 目录树维护入口；本页「管理」指向该能力或说明权限不足。 |

## 6. 功能需求

### 6.1 业务线目录树（FR-CAT-01）

- **结构**：支持多级文件夹节点与叶节点（与原型 `ASSET_TREE_DATA` + `renderTree(..., 'asset')` 一致）；节点可展开/收起（`toggleTreeNode`）。
- **计数**：节点可展示其下资产数量（原型 `countRecordsForTreeNode('asset', path)`，按资产 `catalog` 字段**等于路径或以前缀路径归属**统计）。
- **选中态**：当前选中节点高亮（`tree-node.active`）。
- **验收**：展开/收起无错位；空目录节点计数可为 0 并允许样式区分（`is-empty`）。

### 6.2 目录树与列表联动（FR-CAT-02）

- **规则**：选中某目录节点后，列表**仅展示** `catalog` 等于该节点路径，或位于该路径**子层级**（`catalog.startsWith(path + '/')`）的资产；选中「全部」或等价根态时，不按目录路径过滤。
- **说明**：左侧树与列表的联动应与节点**计数逻辑**一致，避免出现「树显示有子资产但列表为空」的互斥错误。
- **验收**：切换树节点后列表与分页重置为第一页；与关键词、类型、负责人、我的表筛选**同时生效**时为 AND 关系。

### 6.3 面包屑（FR-CAT-03）

- **展示**：反映当前目录上下文（如：全部 / 某域 / …）；与树选中态同步。
- **交互**：「全部」可点击，效果等同于清空目录限定回到根浏览（与原型 `selectAssetNode(null)` 意图一致）。
- **验收**：面包屑与树选中、列表过滤三者一致。

### 6.4 「管理」入口（FR-CAT-04）

- **描述**：面向管理员/治理角色的目录维护入口；无权限时可 toast/文案提示，不报错页。
- **验收**：有权限用户可进入资源管理中的目录管理或等价页面；无权限时提示清晰。

### 6.5 关键词搜索（FR-CAT-05）

- **占位**：与原型一致，如「请输入资产名称/描述关键字」。
- **行为**：输入即过滤列表（实现建议 **debounce**）；与目录、类型等条件组合为 AND。
- **匹配范围**：至少名称、描述/摘要等（与 `matchesAssetSearch` 语义对齐；具体字段以检索服务为准）。
- **验收**：改关键字后列表与分页更新；清空后恢复仅受其它条件约束的结果。

### 6.6 负责人筛选（FR-CAT-06）

- **交互**：下拉触发器、面板内搜索、清空（与原型 `toggleOwnerDropdown('asset')`、`handleOwnerSearch`、`clearOwnerFilter('asset')` 一致）。
- **规则**：筛选当前负责人等于选中人的资产（技术/业务负责人字段并集或产品约定字段，以实现为准）。
- **验收**：选中、清空、搜索列表联动正确。

### 6.7 「我的表」（FR-CAT-07）

- **描述**：快捷限定为**当前登录用户作为负责人**的资产（与原型 `toggleMyData('asset')` / `isRecordOwnedByCurrentUser` 语义一致）。
- **验收**：开关态明确；与其它筛选 AND。

### 6.8 排序（FR-CAT-08）

- **字段**：热度、创建时间、更新时间（与原型 `asset-sort-field` 选项一致）。
- **顺序**：升序/降序可切换（`toggleSortOrder('asset')`）。
- **验收**：排序变化后列表顺序符合字段语义；分页保持当前页策略与产品约定（通常重置到第 1 页）。

### 6.9 类型 Tab（FR-CAT-09）

- **选项**：全部、表、视图、API、报表、指标、标签（与原型 `switchAssetType` 一致）。
- **验收**：切换 Tab 后列表仅含对应 `type`；选中态清晰。

### 6.10 标签筛选（FR-CAT-10）

- **描述**：入口按钮与资源发现一致；若一期仅占位，须在交付清单中标注**假交互下线**或二期排期。
- **验收**：已接入时，筛选与列表、计数一致；未接入时不误导用户（禁用或隐藏）。

### 6.11 资产卡片（FR-CAT-11）

- **点击区域**：整张卡片进入详情 `openDetail('asset', id)`；「申请权限」等行内按钮 `stopPropagation` 避免误进详情（与原型一致）。
- **建议展示字段**：类型图标、技术名（表/视图可拆分库前缀与对象名）、复制技术名、类型标签、状态（如正常/已废弃）、少量业务标签、**申请权限**、来源（短展示）、**业务目录**、近 30 天查询量/浏览量（或等价热度）、描述摘要、技术/业务负责人、创建/更新时间。
- **验收**：字段与详情页无冲突；已废弃等状态样式可辨。

### 6.12 分页（FR-CAT-12）

- **描述**：底部分页器，切换页码保留当前筛选与目录上下文（与原型 `renderPagination('asset-pagination', ...)` 行为一致）。
- **页大小**：原型为每页 8 条；实现可配置，但需在界面或配置中固定对外口径。

### 6.13 空状态（FR-CAT-13）

- **描述**：无数据时展示空状态（原型「暂无数据」）；提示用户可调整目录、关键字或类型。
- **验收**：无崩溃、无残留旧数据。

## 7. 数据与权限

- 列表数据源仅为**已上架、对外可见**的资产集合；具体是否包含「已废弃」等状态由总册枚举与产品策略决定，但须在卡片上**明示状态**。
- 用户仅能看见**有浏览权限**的资产；无权限对象不出现在列表（或脱敏策略按平台统一规范）。
