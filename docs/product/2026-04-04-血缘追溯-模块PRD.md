## 2. 模块定位与背景

### 2.1 一句话定位

**血缘追溯**提供**表级（及展示字段级关联）**的上下游图谱：用户在详情页「血缘关系」Tab 或展开的全屏血缘视图中缩小/放大视角、按层级与方向过滤、点击节点查看摘要并跳转到资产详情，必要时进入**血缘修正**流程。

### 2.2 要解决的问题

- 消费侧需要理解「数据从哪来、到哪去」，支撑影响分析与取数前的信任建立。
- 详情 Tab 空间有限，需要**全屏血缘**承载复杂图与抽屉详情。
- 字段级血缘需要**可发现、可高亮**（选中字段后展示跨表关联字段）。

### 2.3 非目标（本模块不做）

- 血缘**自动采集**规则、SQL 解析、调度依赖推断的技术实现细则（归属数据平台/元数据管线）。
- 图数据库选型、OLAP 存储结构与增量计算方案（技术方案文档）。
- 若「修正血缘」弹窗仍为原型占位，**真实持久化、审批、审计对接**以单独迭代/接口文档为准（设计参考见上文手动编辑说明）。

## 3. 目标与成功标准

### 3.1 产品目标

- 用户能在 **3 个控件内**完成「方向 + 层级 + 居中/刷新」组合，看清 N 跳内的上下游。
- 点击任意节点可打开**右侧抽屉**，并一键跳转统一详情页（资产维度与当前节点一致）。
- 画布在常规数据量下**可平移、可缩放**，边线与节点对齐可读。

### 3.2 成功指标（可后续量化）

- 全屏血缘入口点击率（从详情 Tab 进入）、平均停留时长。
- 血缘修正入口点击量、修正工单完成率（接入真实流程后）。

## 4. 用户与场景

### 4.1 主要用户

| 角色 | 典型诉求 |
|------|----------|
| 数据分析师 | 看上游贴源、下游应用表，判断是否可引用 |
| 数据开发 | 排查字段血缘、定位作业影响面 |
| 治理/数仓 | 发现错误边或缺失边，发起修正 |

### 4.2 典型场景（用户故事摘要）

1. 从详情页点「全屏血缘」，以当前对象为**中心节点**打开 `page-lineage`。
2. 从一级菜单进入血缘追溯，默认落在平台配置的**示例中心**或最近浏览对象（默认落在平台配置的示例中心或最近浏览对象）。
3. 切换「只看上游 / 只看下游」排查单向依赖。
4. 展开节点字段列表，点选某字段，观察与其它表字段的**关联高亮**。
5. 在抽屉「血缘关系」中点击上下游对象，**切换中心节点**重绘子图。

## 6. 功能需求

### 6.1 进入与上下文（FR-LIN-01）

- **入口**：详情页血缘 Tab **全屏血缘**按钮（打开血缘追溯页）。
- **参数**：对象 id 为业务对象标识；字段名可选，用于字段级高亮初始状态。
- **返回**：顶栏「← 返回」恢复进入前页面。
- **验收**：从详情进入时中心节点与当前浏览对象一致；无 id 时默认策略明确且不白屏。

### 6.2 顶栏 — 范围与层级（FR-LIN-02）

- **血缘范围**：下拉选项 **全部 / 只看上游 / 只看下游**，影响可见节点与边集合。
- **层级**：**1～5 级**，在范围内扩展中心节点的多跳邻居；按中心节点向外递减展开。
- **验收**：切换后立即重绘；与「展开徽章」手动展开组合时无死循环或边重复混乱（边去重逻辑保留）。

### 6.3 顶栏 — 修正与视图操作（FR-LIN-03）

- **修正血缘**：打开血缘编辑弹窗；无权限时提示或隐藏（权限矩阵由治理侧定义）。
- **全屏**：整页容器进浏览器全屏，再次操作退出。
- **回到中心点**：将视图变换重置并以**中心节点**对齐视口。
- **刷新**：范围、层级、展开集合、字段分页、选中与高亮等恢复为初始态并重新布局。
- **验收**：全屏态下图谱仍可操作；刷新后状态与文案一致。

### 6.4 画布 — 布局与表级边（FR-LIN-04）

- **布局**：按列放置上游（左）、中心、下游（右）；列内垂直堆叠，间距与节点高度（含字段区展开）动态计算。
- **边**：表级依赖用 SVG 折线/曲线连接节点锚点；随平移缩放变换。
- **验收**：中心节点有「中心节点」标识；选中节点有选中样式。

### 6.5 画布 — 平移与缩放（FR-LIN-05）

- **平移**：在画布空白处拖拽（避开节点与展开徽章）；鼠标样式 `grab` / `grabbing`。
- **缩放**：滚轮缩放，以指针位置为锚点调整平移（避免跳变）。
- **验收**：与节点点击、字段点击不冲突；触控板/滚轮需阻止默认滚动行为，避免页面整体滚动。

### 6.6 节点 — 展开与字段区（FR-LIN-06）

- **上下游展开徽章**：节点在允许展开的方向上展示数量角标，点击直接切换展开/收起该方向的子树。
- **字段区**：可折叠；展开后展示字段行（名称、类型），支持字段分页（每页条数可配置）。
- **指标/标签**：无内联字段列表时隐藏字段区或仅展示空态。
- **验收**：折叠/展开触发重新布局，边线跟随更新。

### 6.7 字段级关联与高亮（FR-LIN-07）

- **激活**：点击字段行设置当前激活字段，同节点及关联边上的字段行高亮。
- **数据**：字段级边来源以实现为准；需与表级边区分图层或样式（若产品需要图例说明）。
- **验收**：激活后用户可识别「哪些字段参与映射」；再次点击或刷新可清除状态。

### 6.8 右侧抽屉（FR-LIN-08）

- **打开**：点击节点头部选中节点并打开抽屉；关闭按钮收起，关闭后可选是否清除选中高亮。
- **Tab**：
  - **基本信息**：摘要字段 + 「查看资产详情」「修正血缘」按钮（点击进入详情页）。
  - **字段信息**：字段搜索、分页列表；点击行可激活字段（与画布联动）。
  - **血缘关系**：展示上下游列表摘要，点击项**跳转并以该节点为中心**重绘。
- **对象类型差异**：右侧抽屉的 Tab 构成按节点类型差异化展示：

  | 节点类型 | 基本信息 | 字段信息 | 血缘关系 | 特殊 Tab |
  |----------|:--------:|:--------:|:--------:|----------|
  | 表 | ✓ | ✓（字段列表+激活联动） | ✓ | — |
  | 视图 | ✓ | ✓ | ✓ | — |
  | 指标 | ✓ | — | ✓ | — |
  | 标签 | ✓ | — | ✓ | — |
  | API | ✓ | — | ✓ | 参数信息（请求/响应参数列表） |
  | 报表 | ✓ | — | ✓ | — |

  - 表/视图有字段信息 Tab，支持字段搜索与画布联动；其他类型无字段信息 Tab。
  - API 类型额外展示「参数信息」Tab，展示请求参数与响应参数列表（与详情页参数信息 Tab 内容一致）。
  - 指标/标签/报表无字段信息 Tab，血缘关系 Tab 中的上下游节点可能包含表、API 等不同类型，点击可跳转对应节点。
- **验收**：Tab 切换不丢失当前节点上下文；跳转后面包屑/中心标识正确更新。

### 6.10 血缘编辑（FR-LIN-10）

血缘编辑用于**人工补录或修正**系统自动采集的血缘关系，覆盖「新增边」「删除边」「字段映射修正」三类操作。

#### 6.10.1 节点类型与血缘约束

血缘节点分为两类，建立关系时的约束不同：

| 节点类型 | 包含对象 | 建立血缘时的约束 |
|----------|----------|------------------|
| 表级节点 | 表 / 视图 / API | 与其他表级节点建立关系时，必须同时配置字段级映射（必填，不可跳过） |
| 字段级节点 | 标签 / 指标 | 与表级节点建立关系时，需选择表再选择表下的具体字段；与字段级节点建立关系时，直接选择对方节点 |

- 表级节点之间的字段映射支持**多对多**。
- 字段级节点与表级节点的关系支持**一对多**（一个标签/指标可关联多个字段）。
- 字段级节点之间支持多关系（如一个标签下游有多个标签，或一个标签上游是多个指标）。

#### 6.10.2 入口

- **全屏血缘页顶栏**：「编辑血缘」按钮，打开血缘关系管理面板。
- **右侧抽屉 - 基本信息 Tab**：「修正血缘」按钮，以当前选中节点为默认主体打开面板。
- **权限控制**：仅对象负责人或治理角色可见；无权限用户不展示入口。

#### 6.10.3 血缘编辑管理面板

面板分为两个 Tab：**表级血缘** 和 **字段级血缘**。

**Tab 1：表级血缘**

展示当前对象的所有上下游关系列表，支持筛选（方向/来源/状态）和搜索。

| 列 | 说明 |
|----|------|
| 方向 | 上游 / 下游 |
| 资源名称 | 资源英文名 + 中文名 |
| 类型 | 表 / 视图 / API / 报表 / 指标 / 标签 |
| 来源方式 | 自动采集 / 手动添加 |
| 状态 | 有效 / 已排除 / **待新增** / **待删除** |
| 操作 | 排除 / 删除 / 撤销 |

- 点击「排除」或「删除」：行不立即消失，状态变为**待删除**（橙色标签），操作变为「撤销」。
- 点击「+ 添加血缘关系」：打开新增子弹窗（见 §6.10.4）。
- 新增的边在列表中以**待新增**（蓝色标签）标注，未提交前可撤销（点击「撤销」直接移除）。

**Tab 2：字段级血缘**

展示当前对象的所有字段级映射关系，支持筛选和搜索。

| 列 | 说明 |
|----|------|
| 方向 | 上游 / 下游 |
| 上游资源.字段 | 上游资源名称 + 字段名 |
| 下游资源.字段 | 下游资源名称 + 字段名 |
| 来源方式 | 自动采集 / 手动添加 |
| 状态 | 有效 / 已排除 / **待新增** / **待删除** |
| 操作 | 排除 / 删除 / 撤销 |

- 字段级映射的待新增/待删除/撤销逻辑与表级一致。
- 通过「+ 添加血缘关系」子弹窗新增的字段映射，在此 Tab 中以**待新增**标注。

#### 6.10.4 新增血缘子弹窗

点击「+ 添加血缘关系」打开子弹窗，流程如下：

1. **选择方向**：添加上游 / 添加下游（单选）
2. **选择资源类型**：表 / 视图 / API / 报表 / 指标 / 标签
   - 当前对象为标签/指标时，资源类型自动锁定为「表」
3. **选择目标资源**：按资源类型筛选后搜索选择
4. **配置字段映射**（必填，表级节点之间必须完成）：
   - 逐条手动配置：左侧选上游字段，右侧选下游字段，可添加多行
   - 可选：点击「同名字段自动匹配」，系统按同名字段批量生成候选映射，用户可在此基础上调整
   - 字段级节点（标签/指标）与表级节点建立关系时，需选择表下的具体字段
5. **填写修正原因**（可选）
6. 点击「确认添加」：新增的表级边和字段映射分别进入两个 Tab 的列表，均标记为**待新增**

> 表级节点之间新增血缘时，**字段映射为必填**，未配置字段映射不能完成新增。

#### 6.10.5 提交与审批

- 面板底部「提交修正」按钮，有 pending 变更时才可点击，按钮显示变更数量（如「提交修正（3 项变更）」）。
- 提交后走**审批流**，toast 提示「修正申请已提交，审批通过后生效，可在操作记录 - 血缘修正中查看进度」，面板关闭。
- 审批通过后血缘图刷新生效；审批拒绝时操作记录中记录拒绝原因。
- **原型阶段**：提交后直接 mock 成功，不接真实审批流。

#### 6.10.6 操作记录联动

血缘编辑的每次提交均写入详情页「操作记录」Tab 的**血缘修正**子类：

| 字段 | 说明 |
|------|------|
| 操作时间 | 提交时间 |
| 操作人 | 提交人 |
| 修正类型 | 新增上游 / 新增下游 / 删除上游 / 删除下游 / 字段映射修正 |
| 修正内容摘要 | 如「新增上游：ods_order_info → dwd_order_detail（3 个字段映射）」 |
| 审批状态 | 审批中 / 已通过 / 已拒绝 |

#### 6.10.7 本期范围说明

- 本期实现**表级血缘与字段级血缘的新增与删除**，含待新增/待删除中间态与撤销能力。
- 新增血缘时字段映射为必填（表级节点之间），同名字段自动匹配为可选辅助功能。
- 真实持久化、审批流对接、审计日志以后续迭代为准。

## 7. 权限与安全

- **只读用户**：可浏览其有权限的对象及血缘子图；无权限的节点应裁剪或脱敏（策略与元数据权限一致）。

## 8. 验收标准（模块级检查清单）
