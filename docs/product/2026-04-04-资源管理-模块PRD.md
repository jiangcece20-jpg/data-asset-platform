## 2. 模块定位与背景

### 2.1 一句话定位

**资源管理**面向数仓工程师与平台运营，提供**资源治理后台**：在工作台掌握待办与动态，在资源列表按状态治理单条与批量数据，在目录管理中维护业务目录树并将资源归属到叶子目录。

### 2.2 要解决的问题

- 新采集资源如何**补齐元数据**、何时可上架。
- **状态流转**（待维护、已上架、不上架等）的可视化与操作入口集中。
- **目录归属**与「资源发现 / 资产目录」展示一致，避免用户侧与治理侧口径分裂。
- 关键治理动作可追溯（操作记录）。

### 2.3 非目标（本模块本期不展开或仅占位）

- 与 `2026-03-26-原型改版清单` 中规划的独立 **「上下架管理 / 标签管理 / 异常处理」等 Tab 全量模块**：原型当前为**工作台 + 资源列表 + 目录管理**骨架；下列需求含 **P1 能力** 时在 §11 单独标明。
- 真实审批流引擎对接（见 `2026-04-03-审批流程设计.md` 等，以对接文档为准）。
- 替换权限中心（Permission Center）的审批操作台职责。

## 3. 目标与成功标准

### 3.1 产品目标

- 数仓工程师进入资源管理后 **10 秒内**理解「我负责多少资源、待办在哪」。
- 资源列表支持按状态 Tab + 工具栏条件快速缩小范围，**行级操作**与当前状态匹配（无非法操作）。
- 目录管理支持树浏览、节点详情、新增一级目录；与资源侧「目录归属」字段同源。

### 3.2 成功指标（可选）

- 待维护资源处理时长、上架转化率。
- 批量操作使用率、目录变更次数。
- 操作记录查询频次（审计）。

## 4. 用户、角色与权限

| 角色 | 主要职责 |
|------|----------|
| 数仓工程师 | 维护自己负责的资源信息、发起上架/不上架、批量交接、目录调整（在权限范围内） |
| 业务线 / 平台管理员 | 审核上架、驳回、查看全局治理数据（具体以权限模型为准） |

**权限原则**：列表可见范围、行操作、批量操作、目录增删改均需与后端权限策略一致；原型中「仅我负责」为筛选能力，**不等于**权限控制本身。

## 6. 功能需求 — 工作台

### 6.1 统计卡片（FR-MGT-01）

- **我负责的资源**：展示数量及副文案（如按状态拆分）；点击可跳转**资源列表**并带上「仅我负责」或等价筛选（与原型一致）。
- **待处理事项**：聚合待维护、待审核、异常等（口径以实现为准）；点击可滚动至待办区或跳转列表 Tab。
- **本周变更**：展示变更条数或趋势（可选）。
- **资源健康度**：可用占位或简单公式（如已补齐描述占比）；进度条与数值一致。

### 6.2 待办事项列表（FR-MGT-02）

- 展示待处理条目（类型、资源名、跳转「去处理」至列表或详情）。
- 支持空状态。

### 6.3 今日操作记录（FR-MGT-03）

- 时间线或表格简版：操作人、资源、动作、结果、时间。
- 与「操作记录抽屉」数据来源可一致或子集。

### 6.4 我的申请（进行中）（FR-MGT-04）

- 展示当前用户未完成申请摘要；「查看全部」跳转「我的」页或统一申请中心（与原型一致）。

## 7. 功能需求 — 资源列表

### 7.1 状态 Tab（FR-MGT-10）

- **Tab**：全部、待维护、已上架、不上架、异常（带角标数量，与原型一致）。
- **规则**：与资源 `status` 字段映射；「异常」可来自独立异常表或打标（实现定义）。
- **验收**：切换 Tab 后表格数据源与 Tab 语义一致；角标与列表可解释。

### 7.2 批量操作栏（FR-MGT-11）

- 未选择时展示「未选择」，操作按钮禁用或隐藏。
- 多选后展示已选数量与可用批量操作（如批量上架、批量不上架、批量改目录、交接负责人等，**以权限与状态机为准**）。
- **取消选择**清空勾选。

### 7.3 工具栏（FR-MGT-12）

- **关键词搜索**：按资源名称过滤当前 Tab 数据集。
- **类型下拉**：全部类型 / 表 / API / 报表 / 标签 / 指标（与枚举一致）。
- **未归属**：勾选后仅展示无目录归属（或业务目录为空）的资源。
- **仅我负责**：勾选后仅展示当前用户为技术或业务负责人的资源（口径与资源发现「我的数据」对齐）。
- **操作记录**：打开侧抽屉，展示治理操作日志（字段见 §7.6）。

### 7.4 表格列（FR-MGT-13）

至少包含：

- 多选框、资源名称（可含跳转详情链接）、类型、平台/来源、**当前状态**、目录归属、资源描述、技术负责人、业务负责人、标签、更新时间、**操作**（ sticky 列）。

**操作列**按状态展示不同动作（示例，以实现为准）：

- 待维护：编辑、上架、不上架等。
- 已上架：查看详情、下架（若产品允许）等。
- 不上架：恢复待维护或重新评估等。

禁止在**资源发现**出现的主操作在此处出现。

### 7.5 分页（FR-MGT-14）

- 列表底部分页；切换 Tab 或筛选后回到合理页码。

### 7.6 操作记录抽屉（FR-MGT-15）

- 从右侧滑出；字段：操作、资源名称、操作人、结果、时间。
- 点击遮罩或关闭按钮关闭。

## 8. 功能需求 — 目录管理

### 8.1 布局（FR-MGT-20）

- 左侧：**目录结构树** + 「新增一级」按钮。
- 右侧：未选节点时空状态；选中节点后展示节点详情与可执行操作（编辑、删除、新增子级、挂载资源等——与原型弹窗一致）。

### 8.2 树操作（FR-MGT-21）

- 支持展开/折叠、选中高亮。
- **新增一级**：打开弹窗，填写父节点（一级时无父或根）、目录名称、描述等。
- 删除需二次确认；删除前校验是否存在子节点或挂载资源（规则由后端定义）。

### 8.3 与资源的关系

- 资源「目录归属」可选值为目录树**叶子节点**（或产品允许任选层级时以规则为准，见历史边界文档）。
- 批量迁移、挂载未归属等资源列表动作与目录树数据一致刷新。

---

## 9. 状态与动作（逻辑对齐）

与总册一致，资源侧典型状态包括：**待维护、待审核、已上架、已驳回、不上架、已下架**等。本期原型 Tab 为子集时，在实现文档中标注**映射表**与二期 Tab 扩展计划。

每个动作须满足：

- 前置条件（字段完整度、权限、当前状态）。
- 成功后刷新列表、写操作记录、必要时通知审批人。

---

## 10. 交互与体验

- 表格横向滚动时 **首列（勾选+名称）与末列（操作）** 宜固定（与原型 sticky 一致）。
- 批量操作前**二次确认**（尤其下架、删除目录）。
- 长文案列（描述、标签）支持省略与 hover 或展开。
- 异常 Tab 与列表加载失败需有明确错误提示。
