## 2. 模块定位

### 2.1 一句话定位

**统一详情页**是资产与资源的**同一套详情骨架**：根据对象类型（表/视图/指标等）展示差异化 Tab 与操作区，在固定右侧栏沉淀高频元数据，在主区承载字段/样例/血缘/说明/治理记录等深度内容。

### 2.2 非目标

- 不替代「资源管理」后台列表上的批量治理（批量仍以列表为准）。
- 不在本 PRD 中展开**真实审批流**对接细则（见审批/权限相关文档）。

## 4. 表头区（摘要）

详细字段、样式、hover 定义已从总册迁入本节。

### 4.0 标题结构总览

标题区采用两层结构：

**第一层**
- 英文名 | 中文名 | 来源信息块 | 复制按钮 | 近30天查询量 | 近30天浏览量 | 申请权限按钮 | 收藏按钮

**第二层**
- 信息完善度 | 数据目录 | 数据分类标签

**展示规则**
- 英文名优先展示完整技术名，若存在库名则展示为 `库名.表名`
- 中文名紧随英文名展示，用于业务理解
- 来源信息块展示：来源类型图标 + 来源系统 + 来源实例或来源位置
- 复制按钮仅复制英文名
- 查询量、浏览量与标题保持同一行
- 数据目录支持多目录展示，不同目录使用 `|` 分隔，同一目录内部层级使用 `/` 分隔
- 数据分类使用标签样式，不使用长文本平铺

**视觉规则**
- 数据目录使用浅蓝色链接式文本，目录间分隔符 `|` 使用灰色
- 数据分类使用圆角标签
- 查询量与浏览量图标化显示，尺寸小于主标题

#### 查询量/浏览量 Hover 说明

查询量与浏览量需支持 hover 提示：

| 字段 | 定义 |
|------|------|
| 近30天查询量 | 近30天内用户通过搜索、筛选、目录检索等方式命中该资源并进入详情的次数 |
| 近30天浏览量 | 近30天内详情页被访问、打开或再次查看的次数 |

交互规则：鼠标 hover 时显示深色浮层提示，浮层位于数字上方，离开后自动消失。

模块验收要点：

### 4.1 第一层（FR-DTL-10）

- 技术名（表/视图支持库名与对象名分层展示）、中文名、来源块、复制、近 30 天查询量/浏览量、**申请权限**、**收藏**。

**各类型技术名展示规则：**

| 类型 | 技术名展示方式 |
|------|--------------|
| 表 table | 库名（灰色小字）+ 表名（粗体），支持分行展示 |
| 视图 view | 库名（灰色小字）+ 视图名（粗体），与表一致 |
| 指标 metric | 英文标识符（如 `metric_gmv_daily`），无库名前缀 |
| 标签 label | 英文标识符（如 `label_user_value_tier`），无库名前缀 |
| API api | 接口英文名（如 `api_user_info`），无库名前缀 |
| 报表 report | 报表英文名（如 `rpt_sales_daily`），无库名前缀 |

### 4.2 第二层（FR-DTL-11）

- 信息完善度、数据目录（多目录 `|` 分隔，目录和｜要有颜色区分）、数据分类标签。

【此处需插入原型图：统一详情页 — 表头动作区，展示 view 模式（申请权限/已有权限/申请中状态切换）与 mgmt 模式（上架/下架/编辑等操作按钮）的对比】

### 4.3 按入口来源的动作区（FR-DTL-12）

详情页的动作区按**入口来源**分为两种视图模式，在进入详情页时由调用方传入：

| 视图模式 | 触发入口 | 说明 |
|----------|----------|------|
| 消费视图 | 资源发现、资产目录、全局搜索、血缘图、AI 找数等 | 默认模式 |
| 管理视图 | 资源管理列表 | 负责人/管理员操作入口 |

#### 消费视图

无论是资产还是资源，动作区只展示：

- **申请权限**（三态：申请权限 / 申请中 / 已有权限，与表头权限状态联动）
- **收藏**（与表头收藏按钮联动）

#### 管理视图

动作区根据对象所属域与资源当前状态展示治理操作，**仅对有权限的负责人/管理员可见**：

| 所属域 | 资源状态 | 展示按钮 |
|--------|----------|----------|
| 资源 | 待维护 | 提交上架、不上架 |
| 资源 | 上架审批中 | 撤回上架申请 |
| 资源 | 已上架 | 修改目录、申请下架 |
| 资源 | 下架审批中 | 撤回下架申请 |
| 资源 | 不上架 | 转待维护 |
| 资产 | — | 管理视图下资产侧不展示治理动作（治理操作在资源侧完成） |

> 「提交上架」前需校验：中文名、描述、目录归属、负责人均已填写，否则拦截并引导去完善。

所有治理操作均走审批流，操作结果记录到「操作记录」Tab（FR-DTL-26）。

## 5. Tab 与对象类型矩阵

打开详情时根据对象类型渲染对应的 Tab 集合。

| 类型 | Tab 顺序（key） |
|------|-----------------|
| **表 table** | 字段信息（fields）→ 样例数据（sample）→ 分区信息（partitions）→ 血缘关系（lineage）→ 使用说明（manage）→ DDL变更（ddl）→ 操作记录（logs） |
| **视图 view** | 字段信息（fields）→ 样例数据（sample）→ 血缘关系（lineage）→ 使用说明（manage）→ DDL变更（ddl）→ 操作记录（logs） |
| **指标 metric** | 指标定义（definition）→ 血缘关系（lineage）→ 使用说明（manage）→ 操作记录（logs） |
| **标签 label** | 标签定义（definition）→ 血缘关系（lineage）→ 使用说明（manage）→ 操作记录（logs） |
| **API api** | 参数信息（params）→ 血缘关系（lineage）→ 使用说明（manage）→ 操作记录（logs） |
| **报表 report** | 报表定义（definition）→ 血缘关系（lineage）→ 使用说明（manage）→ 操作记录（logs） |

**说明：**
- 视图（view）无分区信息 Tab，无 DDL 变更 Tab（若视图无分区可不展示）。
- 指标/标签/报表/API 的第一个 Tab 均为「定义类」Tab，key 统一为 `definition`（API 例外，key 为 `params`），渲染内容按类型差异化（见 §6.8）。
- 「操作记录」Tab 在所有类型中均存在，key 统一为 `logs`。

## 6. 主内容 Tab — 需求摘要

### 6.1 字段信息（表/视图）（FR-DTL-20）

> 仅表和视图类型包含此 Tab；指标/标签/API/报表无字段信息 Tab，其结构信息在各自的定义类 Tab（ 指标定义 / 标签定义 / API 参数信息 / 报表定义）中呈现。

- 表格列：序号、字段名、类型、注释、**字段描述**（可编辑）、**安全等级**。
- 字段顺序**严格按数据库返回的原始顺序**展示，不做前端重排（见总册 §26.1）。

**字段描述**
- 字段描述是字段级业务/技术解释信息，区别于"注释"，更强调人工完善后的业务含义与使用口径。
- 技术负责人可直接在字段结构页中修改，修改后提示"字段描述已更新"。

**安全等级**

| 等级 | 分级标准 | 展示规则 |
|------|----------|----------|
| S1 | 公开级 — 非敏感数据 | 正常展示 |
| S2 | 内部级 — 不适合公开，危害较低 | 正常展示 |
| S3 | 秘密级 — 一定量泄漏会造成严重危害 | 标记需加密，样例脱敏 |
| S4 | 绝密级 — 关键敏感数据，少量泄漏即有严重危害 | 标记需加密，样例脱敏 |
| S5 | 重要及核心级 — 核心机密数据 | 标记需加密，样例脱敏 |

- `S3/S4/S5` 在字段行中显示"需加密"标识，等级颜色按风险升高递进。
- 字段表格上方展示字段治理说明卡片，解释安全等级规则与脱敏策略。
- 存在分区字段时：**同一行工具区**内提供「字段信息 | 分区信息」分段 + **字段关键词筛选**（匹配字段名、注释、描述、安全等级文案等）。
- 字段描述变更后持久化并 toast；权限仅数仓/负责人可编辑（规则与后端一致）。

### 6.2 样例数据（FR-DTL-21）

**默认展示**
- 默认加载最新 10 条数据。
- 列顺序与字段信息 Tab 保持一致（按数据库原始顺序）。
- 表头固定，横向超出时支持横向滚动，关键列（序号、字段名）sticky 在左侧。

**S1–S5 脱敏规则**

| 安全等级 | 样例数据展示规则 |
|----------|----------------|
| S1 公开级 | 正常展示原始值 |
| S2 内部级 | 正常展示原始值 |
| S3 秘密级 | 统一显示 `******`，不展示任何原始字符 |
| S4 绝密级 | 统一显示 `******`，不展示任何原始字符 |
| S5 重要及核心级 | 统一显示 `******`，不展示任何原始字符 |

- S3/S4/S5 字段的列头显示「🔒 已脱敏」角标，hover 提示「该字段安全等级为 Sx，样例数据已脱敏」。
- 脱敏在前端渲染层执行，后端接口对 S3+ 字段返回脱敏后的占位值，前端不做二次还原。
- 若整张表所有字段均为 S3+，则样例数据区域展示空状态提示：「该表所有字段均为高敏等级，样例数据不可查看」。

**加载与错误状态**
- 加载中：表格区域展示骨架屏。
- 加载失败：展示「样例数据加载失败，请刷新重试」，提供重试按钮。
- 无数据：展示「暂无样例数据」空状态。

### 6.3 分区信息（FR-DTL-22）

> 仅表类型 Tab 集包含本分页；视图无分区，不展示该 Tab（视图的 Tab 集为：字段信息 → 样例数据 → 血缘关系 → 使用说明 → DDL变更 → 操作记录）。

**列定义**

| 列名 | 说明 |
|------|------|
| 分区键 | 分区字段名，支持多级分区（如 `dt / hour`） |
| 分区值 | 该分区的具体值（如 `2026-04-01`） |
| 数据量（行数） | 该分区的估算行数，来源于元数据统计 |
| 存储大小 | 该分区占用存储空间（单位自适应：KB / MB / GB） |
| 创建时间 | 分区创建时间 |
| 最近更新时间 | 分区最近一次写入/更新时间 |

**搜索与筛选**
- 工具栏提供分区值关键词搜索框，支持模糊匹配分区值。
- 默认按「最近更新时间」倒序排列，支持点击列头切换排序。
- 默认展示最近 100 个分区，超出时底部提示「仅展示最近 100 个分区，如需查看更多请使用即席查询」。

**空状态**
- 表无分区时：展示「该表暂无分区信息」，不展示表格。

### 6.4 血缘关系（FR-DTL-23）

【此处需插入原型图：统一详情页 — 血缘关系Tab，展示页内嵌入式血缘小画布及「全屏血缘」入口按钮】

**布局**
- 血缘关系 Tab 下采用全宽布局，隐藏右侧固定信息区，图谱占满整个内容区域。

**图谱控制栏**（位于画布右上角浮层）

| 控件 | 功能 |
|------|------|
| 范围选择 | 切换「上游 / 下游 / 全部」血缘方向 |
| 层级选择 | 选择展开层数（1 / 2 / 3 / 全部），默认 2 层 |
| 居中 | 将当前节点重新居中对齐 |
| 刷新 | 重新拉取血缘数据并重绘图谱 |
| 全屏 | 进入全屏模式，全屏下再次点击退出；全屏时 ESC 也可退出 |
| 修正入口 | 跳转至血缘修正页面（仅负责人/管理员可见） |

**节点交互**
- 点击节点弹出节点详情抽屉（从右侧滑入），抽屉包含「基本信息 / 字段信息 / 血缘关系」三个子 Tab。
- 节点详情抽屉使用 fixed 定位挂载于 document.body，z-index 高于页面 header，确保不被遮挡。
- 抽屉打开时图谱不关闭，支持在图谱中继续点击其他节点切换抽屉内容。

**空状态**
- 无血缘数据时：展示「暂无血缘信息，如需补充请使用修正入口」。

### 6.5 使用说明 / 维护（FR-DTL-24）

**富文本编辑区**
- 支持标准富文本能力：加粗、斜体、有序/无序列表、标题层级（H2/H3）、行内代码、代码块、超链接、图片上传。
- 编辑权限：仅技术负责人或业务负责人可编辑；其他用户只读展示。
- 自动保存：内容变更后 3 秒无操作自动保存，保存成功 toast 提示「使用说明已保存」。
- 字数上限：10,000 字，超出时拦截并提示。

**负责人字段**

| 字段 | 说明 |
|------|------|
| 技术负责人 | 可多选，支持按姓名/工号搜索，最多 5 人 |
| 业务负责人 | 可多选，支持按姓名/工号搜索，最多 5 人 |

- 负责人变更后立即生效，同步更新右侧基础信息区对应字段。
- 变更记录写入「操作记录」Tab（FR-DTL-26）。

**维护备注**
- 独立于富文本的纯文本备注区，用于记录临时维护说明（如「2026-04-01 数据延迟，预计次日修复」）。
- 字数上限：500 字。
- 展示时间戳与最后修改人。

**空状态**
- 未填写时展示「暂无使用说明，点击编辑补充」引导文案（仅负责人可见编辑入口）。

### 6.6 DDL 变更（FR-DTL-25）

**列表定义**

| 列名 | 说明 |
|------|------|
| 变更时间 | DDL 变更发生的时间，精确到秒 |
| 变更类型 | 枚举：新增字段 / 删除字段 / 修改字段类型 / 修改字段注释 / 修改表属性 / 其他 |
| 变更描述 | 简要描述本次变更内容（如「新增字段 user_id bigint」） |
| 操作人 | 执行 DDL 的账号，若为系统自动变更则显示「系统」 |
| 操作 | 「查看 SQL」按钮 |

- 默认按变更时间倒序排列，不支持手动排序。
- 默认展示最近 50 条，支持分页（每页 20 条）。

**展开 SQL**
- 点击「查看 SQL」在当前行下方内联展开 SQL 代码块（行内展开，不弹窗）。
- SQL 代码块支持语法高亮（SQL 语法）、一键复制。
- 再次点击「收起 SQL」折叠代码块。

**空状态**
- 无变更记录时：展示「暂无 DDL 变更记录」。

### 6.7 操作记录（FR-DTL-26）

- 二级分类（如上架/普通变更/血缘修正）见总册相关章节；与资源管理「操作记录」数据源可对齐或子集。

### 6.8 定义类 Tab — 按对象类型差异化（FR-DTL-27）

四种类型的第一个 Tab 均为「定义」，内容结构各不相同：

#### 指标定义（metric）

| 区块 | 字段 |

## 8. 权限申请与收藏

### 8.1 申请权限

**按钮三态**

| 状态 | 展示样式 | 触发条件 |
|------|----------|----------|
| 申请权限 | 主色按钮，可点击 | 用户无权限且无在途申请 |
| 申请中 | 灰色 chip，不可点击 | 已提交申请，审批未完成 |
| 已有权限 | 绿色 chip，不可点击 | 审批通过或本身已有权限 |

- 点击「申请权限」后，将当前对象加入「我的」模块中的申请单；若该对象已在申请单中，则提示已加入，不重复添加。
- 加入申请单后，详情页可继续停留当前页面浏览；当用户点击「去申请单提交」时，应直接进入「我的 > 申请单」，不在详情页内展开完整申请单填写流程。
- 「加入申请单」与「已提交审批中」属于两个不同阶段：前者表示资产已进入待提交购物车，后者表示用户已在「我的 > 申请单」中完成提交且审批尚未结束。原型展示可合并为同一条申请链路提示，但产品口径上需区分待提交与审批中。
- 审批通过后，详情页内同步更新以下两处，无需用户刷新页面：
  - 表头动作区「申请权限」按钮 → 变为「已有权限」chip（不可点击）
  - 右侧基础信息区「我的权限」字段：「申请中」→「已有权限」
- 审批拒绝或申请被撤回后，上述两处回退为「申请权限」可点击状态。

**联动范围**：状态变更同步至申请单、「我的-我申请的」、资源发现卡片等相关入口，各入口保持一致。

### 8.2 收藏

- 点击收藏按钮立即切换收藏/取消收藏状态，无需确认框。
- 收藏状态与「我的」模块中的收藏列表实时同步。
- 表头收藏按钮与动作区收藏按钮（如同时存在）状态联动。

## 10. 用户故事

### Story 1：理解资源

作为普通数据用户，我想在详情页快速看到标题、来源、目录、分类、热度和说明，以便快速判断该资源是否值得使用。

验收标准：
- 标题区能同时展示英文名、中文名和来源信息
- 支持复制英文名
- 支持查看查询量和浏览量定义（hover 提示）

### Story 2：申请权限

作为普通数据用户，我想在详情页发起权限申请，以便申请使用该资源。

验收标准：
- 点击「申请权限」后，当前对象加入申请单
- 重复加入时不重复添加，并给出提示
- 用户可继续停留详情页，也可通过「去申请单提交」直接进入「我的 > 申请单」
- 用户在「我的 > 申请单」中统一填写理由并提交申请
- 提交后，该对象进入审批中状态；审批通过或驳回后再同步更新详情页状态

### Story 3：字段治理

作为技术负责人，我想维护字段描述和安全等级，以便提升资源可理解性与治理准确性。

验收标准：
- 字段结构中存在字段描述输入框
- 字段结构中存在安全等级标签
- S3 及以上字段显示需加密标识

### Story 4：安全脱敏

作为平台用户，我希望高敏字段的样例数据自动脱敏，以便在预览时避免敏感信息泄露。

验收标准：
- S3/S4/S5 字段样例数据统一显示 `******`
- S1/S2 字段正常显示
