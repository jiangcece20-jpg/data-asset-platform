## 2. 模块定位

### 2.1 一句话定位

**统一详情页**是资产与资源的**同一套详情骨架**：根据对象 `type` 与 `domain`（资产/资源）展示差异化 Tab 与操作区，在固定右侧栏沉淀高频元数据，在主区承载字段/样例/血缘/说明/治理记录等深度内容。

### 2.2 非目标

- 不替代「资源管理」后台列表上的批量治理（批量仍以列表为准）。
- 不在本 PRD 中展开**真实审批流**对接细则（见审批/权限相关文档）。

## 3. 进入与上下文

### 3.1 入口（FR-DTL-01）

- 从资源发现卡片、资产目录、资源管理列表、全局搜索命中项等进入。
- 路由需携带：`对象 id`、`domain`（`asset` | `resource`）或等价参数，用于拉取详情与决定按钮显隐。

### 3.2 顶栏（FR-DTL-02）

- **返回**：回到上一浏览位置（或指定 fallback 页）。
- **面包屑**：与平台导航一致，可点击回到上级列表/检索。

## 4. 表头区（摘要）

详细字段、样式、hover 定义见总册 **§9**。模块验收要点：

### 4.1 第一层（FR-DTL-10）

- 技术名（表/视图支持库名与对象名分层展示）、中文名、来源块、复制、近 30 天查询量/浏览量、**申请权限**、**收藏**。

### 4.2 第二层（FR-DTL-11）

- 信息完善度、数据目录（多目录 `|` 分隔，目录和｜要有颜色区分）、数据分类标签。

### 4.3 按 domain 的动作区（FR-DTL-12）

详情页的动作区按**入口来源**分为两种视图模式，在 `openDetail` 时由调用方传入：

| 视图模式 | 触发入口 | 说明 |
|----------|----------|------|
| `browse`（消费视图） | 资源发现、资产目录、全局搜索、血缘图、AI 找数等 | 默认模式 |
| `mgmt`（管理视图） | 资源管理列表 | 负责人/管理员操作入口 |

#### browse 模式（消费视图）

无论 domain 是 asset 还是 resource，动作区只展示：

- **申请权限**（与 FR-DTL-10 中的权限状态联动，三态：申请权限 / 申请中 / 已有权限）
- **收藏**（与 FR-DTL-10 中的收藏按钮联动）

#### mgmt 模式（管理视图）

动作区根据 `domain` 和资源当前状态展示治理操作，**仅对有权限的负责人/管理员可见**：

| domain | 资源状态 | 展示按钮 |
|--------|----------|----------|
| resource | 待维护（maintain） | 提交上架、不上架 |
| resource | 审核中（reviewing） | 撤回申请 |
| resource | 已上架（published） | 申请下架 |
| resource | 申请下架中（unlisting） | 撤回下架申请 |
| resource | 不上架（no-list） | 转待维护 |
| asset | — | 管理视图下资产侧不展示治理动作（治理操作在资源侧完成） |

> 「提交上架」前需校验：中文名、描述、目录归属、负责人均已填写，否则拦截并引导去完善。

所有治理操作均走审批流，操作结果记录到「操作记录」Tab（FR-DTL-26）。

## 5. Tab 与对象类型矩阵

打开详情时根据 `getDetailObjectType(record)` 渲染 Tab 集合（与原型 `getDetailTabConfig` 一致）。

| 类型 | Tab 顺序（key） |
|------|-----------------|
| **表 table** | 字段信息（fields）→ 样例数据（sample）→ 分区信息（partitions）→ 血缘关系（lineage）→ 使用说明（manage）→ DDL变更（ddl）→ 操作记录（logs） |
| **视图 view** | 字段信息（fields）→ 样例数据（sample）→ 血缘关系（lineage）→ 使用说明（manage）→ DDL变更（ddl）→ 操作记录（logs） |
| **指标 metric** | 指标定义（definition）→ 血缘关系（lineage）→ 使用说明（manage）→ 操作记录（logs） |
| **标签 label** | 标签定义（definition）→ 血缘关系（lineage）→ 使用说明（manage）→ 操作记录（logs） |
| **API api** | 参数信息（params）→ 血缘关系（lineage）→ 使用说明（manage）→ 操作记录（logs） |
| **报表 report** | 报表定义（definition）→ 血缘关系（lineage）→ 使用说明（manage）→ 操作记录（logs） |

**说明：**
- 视图（view）无分区信息 Tab，无 DDL 变更 Tab（若视图无分区可不展示）。
- 指标/标签/报表/API 的第一个 Tab 均为「定义类」Tab，key 统一为 `definition`（API 例外，key 为 `params`），渲染内容按类型差异化（见 §6.8）。
- 「操作记录」Tab 在所有类型中均存在，key 统一为 `logs`。

## 6. 主内容 Tab — 需求摘要

### 6.1 字段信息（表/视图）（FR-DTL-20）

- 表格列：序号、字段名、类型、注释、**字段描述**（可编辑）、**安全等级**（总册 §15）。
- 存在分区字段时：**同一行工具区**内提供「字段信息 | 分区信息」分段 + **字段关键词筛选**（匹配字段名、注释、描述、安全等级文案等）。
- 字段描述变更后持久化并 toast；权限仅数仓/负责人可编辑（规则与后端一致）。

### 6.2 样例数据（FR-DTL-21）

- 默认条数、表头与 **S1–S5 脱敏规则** 见总册 **§14**。

### 6.3 分区信息（FR-DTL-22）

- 列与搜索见总册 **§16**；仅表类型 Tab 集包含本分页（视图若无分区可不展示该 Tab，已由矩阵约束）。

### 6.4 血缘关系（FR-DTL-23）

- 页内图谱：范围、层级、居中、刷新、全屏、修正入口等见总册 **§17**。
- **全屏血缘**下隐藏右侧固定信息区。

### 6.5 使用说明 / 维护（FR-DTL-24）

- 富文本能力、负责人字段、维护备注等见总册 **§13** 及原型「使用说明」Tab。

### 6.6 DDL 变更（FR-DTL-25）

- 列表、展开 SQL 见总册 **§18**。

### 6.7 操作记录（FR-DTL-26）

- 二级分类（如上架/普通变更/血缘修正）见总册相关章节；与资源管理「操作记录」数据源可对齐或子集。

### 6.8 定义类 Tab — 按对象类型差异化（FR-DTL-27）

四种类型的第一个 Tab 均为「定义」，内容结构各不相同：

#### 指标定义（metric）

| 区块 | 字段 |
|------|------|
| 指标概览（Hero Card） | 中文名、英文名、摘要描述、统计粒度、统计周期、更新频率、最近示例值 |
| 口径说明 | 计算口径（富文本）、适用场景列表 |
| 使用提醒 | 注意事项列表、最近一个周期示例值 |

#### 标签定义（label）

| 区块 | 字段 |
|------|------|
| 基础信息 | 标签编码、标签类型（枚举/数值/布尔等）、标签维度（打标对象）、平台归属、更新方式、更新频率 |
| 值域说明 | 枚举值列表（值 + 含义 + 标签）或数值范围说明 |
| 覆盖率 | 覆盖率百分比 |
| 样本数据 | 示例用户 ID + 标签值（脱敏展示） |
| 使用提醒 | 注意事项列表 |

#### API 参数信息（api）

| 区块 | 字段 |
|------|------|
| 接口概览（Hero Card） | 中文名、HTTP 方法（GET/POST 等）、摘要描述、请求路径、服务归属、鉴权方式、版本号 |
| 请求参数表 | 参数名、类型、是否必填、说明、示例值 |
| 返回字段表 | 字段名、类型、说明、示例值 |
| 调用说明 | 注意事项列表（限流、鉴权、超时等） |

#### 报表定义（report）

| 区块 | 字段 |
|------|------|
| 报表概览 | 报表名称（中文名 + 英文名）、描述、所属系统、来源平台、报表目录 |
| 核心内容与分析视角 | 核心指标列表、分析维度列表 |
| 使用说明 | 适用场景、访问方式 |

> 以上字段以原型 `DETAIL_METRIC_DEFINITIONS` / `DETAIL_LABEL_DEFINITIONS` / `DETAIL_API_DEFINITIONS` / `DETAIL_REPORT_DEFINITIONS` 中的 mock 数据结构为准；字段的最终口径由各域产品确认后更新。

## 8. 权限申请与收藏

见总册 **§12**。本模块补充以下联动要求：

权限审批通过后，详情页内需同步更新以下两处状态，无需用户刷新页面：

- 表头动作区「申请权限」按钮 → 变为「已有权限」chip（不可点击）
- 右侧基础信息区「我的权限」字段：「申请中」→「已有权限」

审批拒绝时，上述两处回退为「待申请」状态。完整联动范围（含卡片、我的申请列表）见总册 **§12.1**。

## 9. 交互与体验

- Tab 切换保留滚动位置策略（可选：每次回到顶部，产品统一即可）。
- 长表格横向滚动时关键列 sticky 策略与资源管理列表对齐设计规范。
- 详情加载失败、无权限访问单独空状态/提示页。
