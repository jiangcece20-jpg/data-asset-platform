## 2. 模块定位与背景

### 2.1 一句话定位

**审批中心**（工单中心）是全量工单的**查询与审批处理**入口：用户查看所有类型工单（权限申请 + 治理操作申请）的进度与详情，审批人在此处理待办。

### 2.2 与相邻模块的边界

| 能力 | 归属模块 | 说明 |
|------|---------|------|
| 申请单提交（购物车式） | 我的 | 普通用户在个人中心提交权限申请；申请链路深链进入时默认落在「我的 > 申请单」 |
| 权限申请进度（仅权限类） | 我的 | 普通用户在「我申请的」查看权限申请状态，作为个人快捷视图 |
| 已授权资产清单 | 我的 | 普通用户在「我有权限的」查看 |
| 全量工单查询（所有类型） | 审批中心 | 本模块；查看权限申请与治理申请的完整记录 |
| 已授权资产清单 | 我的 | 在「我有权限的」面板中查看 |
| 待我审批 | 审批中心 | 本模块 |
| 审批模板配置 | 审批管理 | 管理员后台 |
| 全平台审批记录 | 审批管理 | 管理员后台 |

### 2.3 非目标

- 替代企业统一身份源账号生命周期管理。
- 细粒度列级/行级策略引擎规则配置（归属安全与数仓平台后台）。
- 承载申请单提交（已迁至「我的」模块）。

## 4. 功能需求

### 4.1 工单查询（FR-PCM-01）

【此处需插入原型图：审批中心「工单查询」— 状态Tab（全部/审批中/已通过/已拒绝/已撤回）+ 申请类型筛选 + 工单列表（含申请类型tag/状态tag/操作列）的完整样式】

合并展示当前用户发起的**所有工单**（权限申请 + 治理操作申请），以**状态 Tab** 切换视图。

- **列表字段**：编号、**申请类型**、资产名称（英文名 + 中文名）、资产类型 tag、申请时间、状态、操作（查看详情 / 撤回）。
- **申请类型枚举**：

| 申请类型 | 说明 |
|---------|------|
| 权限申请 | 申请访问某资产的数据权限 |
| 上架申请 | 申请将资源发布为正式资产 |
| 目录变更 | 申请修改资产所属目录节点 |
| 打标签 | 申请为资产添加或修改标签 |
| 下架申请 | 申请将已上架资产下架 |
| 血缘修正 | 提交血缘图修正内容 |

**筛选**：支持按**申请类型**筛选（全部 / 权限申请 / 上架申请 / 目录变更 / 打标签 / 下架申请 / 血缘修正）；支持资产名称关键词搜索。

**状态 Tab 说明：**

| Tab | 包含状态 | 可用操作 |
|-----|---------|---------|
| 全部 | 所有状态 | 查看详情 |
| 审批中 | 流程进行中，等待审批人处理 | 查看详情、撤回 |
| 已通过 | 审批人已通过，权限/变更已生效 | 查看详情 |
| 已拒绝 | 审批人已拒绝，含驳回原因 | 查看详情、重新申请 |
| 已撤回 | 申请人主动撤回，流程终止 | 查看详情 |

- **详情**：展开单笔工单内各资产子状态、审批意见、驳回原因。
- **操作**：撤回（审批中状态可撤回，撤回后不可恢复）、重新申请（仅已拒绝状态可用；触发后将相关资产重新加入申请单，并回流至「我的 > 申请单」继续提交）。
- **入口关系**：普通用户查看个人权限申请进度时，优先使用「我的-我申请的」；需要查看权限申请与治理申请的全量记录时，进入审批中心「工单查询」。
- **验收**：各申请类型在同一列表中可通过类型 tag 区分；撤回后列表即时更新；重新申请可正确回流至「我的 > 申请单」。

### 4.2 待我审批（FR-PCM-02）

【此处需插入原型图：审批中心「待我审批」— 待审批列表（含申请人/申请类型/资产名/申请理由/通过/拒绝操作按钮）及驳回原因必填弹窗的样式】

仅审批人可见。以**状态 Tab** 切换视图，覆盖所有需要当前用户审批的工单（权限申请 + 治理操作申请）。

- **列表字段**：申请人、申请类型、资产名称、资产类型 tag、权限类型/操作摘要、申请理由、申请时间、状态。

**状态 Tab 说明：**

| Tab | 包含状态 | 可用操作 |
|-----|---------|---------|
| 待审批 | 流程进行中，等待当前用户处理 | 通过、拒绝（驳回时必填原因） |
| 已通过 | 当前用户已审批通过，权限/变更已生效 | 查看详情 |
| 已拒绝 | 当前用户已拒绝，含驳回原因 | 查看详情 |
| 已过期 | 申请超时未处理或权限已到期失效 | 查看详情 |

- **验收**：无审批权限用户不展示此入口或展示空态；各申请类型可通过类型 tag 区分；驳回原因必填校验生效；操作后列表即时更新。

> **说明：** 血缘修正申请（由资源管理/详情页发起的血缘边修正操作）也会进入待我审批队列，申请类型 tag 显示为「血缘修正」，审批通过后血缘图自动更新。
