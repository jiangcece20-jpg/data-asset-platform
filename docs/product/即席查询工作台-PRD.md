# 即席查询工作台 — 产品需求文档（PRD）

> 版本：v1.0  
> 日期：2026-04-04  
> 状态：基于原型梳理  
> 原型文件：`manus_2026-03-30_adhoc-sql.html`（传统工作台）、`prototype/notebook.html`（Notebook 工作台）

---

## 1. 背景与目标

### 1.1 背景

数据资产管理平台已形成完整的"找数据 → 看详情 → 申请权限 → 查询分析"工作链路。当前平台具备资产检索、资产目录、资源发现、资源管理等核心模块，但缺少一个独立的、面向全局的 SQL 查询入口。现有的查询能力散落在详情页的"去查询"快捷操作中，仅服务于单表查询场景。

业务分析人员在日常工作中需要频繁对已授权的数据表进行即时查询与多表关联分析，迫切需要一个集中式的查询工作台。

### 1.2 目标

- 在一级导航中新增"即席查询"入口，提供全局性的 SQL 查询工作台
- 用户可在工作台中浏览并选择已有权限的数据表，编写和执行 SQL
- 支持多表选择与 JOIN 查询
- 支持 SQL 草稿保存与历史查询回溯
- 执行前进行权限前置校验，杜绝越权查询
- 保持与详情页"快捷进入查询"流程的兼容

### 1.3 非目标

- 不支持查询结果导出（仅在线预览）
- 不支持查询结果分享
- 不提供复杂 IDE 功能（多标签页、版本对比、脚本管理等）
- 不自动生成 SQL（系统辅助插入表名/字段名，但不代替用户组装逻辑）

---

## 2. 产品定位

**轻量级 SQL 查询工作台** — 聚焦于降低业务用户对已授权数据的查询门槛，而非面向专业开发者的 IDE。

设计原则：
- 围绕"已授权表"组织，而非空白编辑器
- 以快速分析为核心，不追求复杂开发能力
- 增强辅助但不替用户做 SQL 决策
- 权限校验前置，避免执行后才发现越权

---

## 3. 用户角色

| 角色 | 描述 | 典型场景 |
|------|------|---------|
| 业务分析师 | 日常进行经营数据分析 | 查询订单表、用户画像表进行关联分析 |
| 数据用户 | 偶尔需要查数验证的业务人员 | 从详情页进入查询单张表的数据样例 |
| 数仓开发 | 开发过程中需要快速验证数据 | 多表 JOIN 验证数据一致性 |

## 4. 信息架构与导航

### 4.1 一级菜单结构

```
资产检索 | 资产目录 | 资源发现 | 资源管理 | 即席查询 | 我的 | 权限中心 | 资产分析(占位)
```

"即席查询"位于"资源管理"之后、"我的"之前。

### 4.2 页面体系

|------|------|------|
| SQL 工作台 | `manus.html#page-sql` | 传统三栏布局，从详情页"去查询"进入 |

- **资产检索 / 资产目录 / 资源发现** → 找表、看信息、申请权限

#### 5.2.4 筛选按钮组

#### 5.2.5 表卡片

每张表以卡片形式展示，包含：

| 字段 | 说明 |

"全部"视图下，列表分为两个区域：

### 5.3 中间 SQL 编辑与结果区

工作台的核心区域，包含引用表栏、SQL 编辑器、工具栏和结果区四个部分。

#### 5.3.1 引用表栏

位于编辑器上方，展示当前查询涉及的数据表。

```
┌─────────────────────────────────────────────────────┐
│ 当前引用表                                            │
│ 支持选择多张表进行 JOIN 查询，点击表标签可切换右侧字段区    │
│                                                      │
│ [订单明细宽表 wlyd_mc_beijing.dwd_order_detail ×]     │
│ [用户画像宽表 wlyd_mc_beijing.ads_user_profile ×]     │
└─────────────────────────────────────────────────────┘
```

- 每张引用表显示为蓝色标签（chip），包含中文名、fullName、移除按钮（×）
- 点击标签可切换当前焦点表（高亮为实心蓝色），右侧字段面板联动
- 无引用表时显示"暂未选择数据表"

#### 5.3.2 空状态引导

当未选择任何表且编辑器为空时，显示居中引导：

```
🧠
从左侧选择已有权限的数据表，开始即席查询

你可以先选择一张表查看字段，也可以选择多张表后手写 JOIN。
系统支持保存 SQL 草稿、查看历史查询，并会在执行前校验是否引用了无权限表。
```

#### 5.3.3 SQL 编辑器

- **样式**：深色主题（背景 `#1e1e1e`，文字 `#d4d4d4`），等宽字体（Courier New）
- **最小高度**：160px，占满可用空间
- **占位符**：`-- 先从左侧选择已有权限的表，然后开始编写 SQL` + 示例 SELECT 语句
- **输入监听**：任何内容变更标记 `sqlUnsavedChanges = true`

#### 5.3.4 编辑器头部

编辑器上方有一行信息栏：
- 左侧：标题"SQL 编辑器"
- 右侧：当前草稿名称（如"当前草稿：订单与画像关联分析"或"未保存草稿"）+ "清空"链接按钮

#### 5.3.5 工具栏

编辑器下方的操作按钮行：

| 按钮 | 功能 | 说明 |
|------|------|------|
| ▶ 执行 | 执行当前 SQL | 主按钮（蓝色），触发完整的校验→执行流程 |
| 格式化 | 格式化 SQL | 关键字大写，在 FROM/JOIN/WHERE/GROUP BY/ORDER BY/LIMIT 前换行 |
| 保存草稿 | 保存当前 SQL | 打开草稿保存弹窗 |

工具栏右侧显示执行状态标签：
- 执行中：蓝色背景，"⏳ 执行中..."
- 成功：绿色背景，"✓ 执行成功 · 耗时 0.45s · 返回 100 行"
- 失败：红色背景，错误信息

#### 5.3.6 结果区

执行成功后在编辑器下方展示：
- **结果表格**：标准表头 + 数据行，白色背景，横向可滚动
- **结果信息**：底部显示"共 100 行（仅展示前 8 行预览）· 涉及 N 张表 · 不支持导出"

### 5.5 SQL 草稿管理

允许用户将编写中的 SQL 保存为草稿，后续继续编辑。

#### 5.5.1 保存草稿弹窗

点击"保存草稿"按钮打开模态框：

| 字段 | 说明 |
|------|------|
| 草稿名称 | 必填，输入框。编辑已有草稿时预填当前名称；新建时为空，保存时默认"即席查询草稿-N" |
| 当前引用表 | 展示区，显示已选中表的 fullName 标签 |
| 草稿预览 | 展示区，显示当前 SQL 内容（等宽字体，只读） |

#### 5.5.2 保存逻辑

- **首次保存**（`sqlCurrentDraftId` 为空）：创建新草稿，生成新 ID
- **再次保存**（`sqlCurrentDraftId` 已有值）：覆盖更新已有草稿
- 保存内容包括：草稿名称、SQL 文本、引用表列表、保存时间
- 保存后重置 `sqlUnsavedChanges`

#### 5.5.3 草稿数据结构

```
{
  id: "draft-1",
  name: "订单与画像关联分析",
  sql: "SELECT o.user_id, ...",
  tables: ["dwd_order_detail", "ads_user_profile"],
  updatedAt: "2026-03-30 10:12:33"
}
```

### 5.6 历史查询

记录已执行的查询供后续回溯复用。

#### 5.6.1 历史查询弹窗

点击"历史查询"按钮打开模态框，分为左右两栏：

```
┌──────────────────────────────────────────────┐
│              历史查询与草稿                      │
├───────────────────┬──────────────────────────┤
│ 草稿箱             │ 历史查询                  │
│                   │                          │
│ ┌───────────────┐ │ ┌────────────────────┐   │
│ │ 草稿名称       │ │ │ 查询摘要            │   │
│ │ 保存时间/涉及表│ │ │ 执行时间/涉及表      │   │
│ │ [载入草稿]     │ │ │ [回填 SQL]          │   │
│ └───────────────┘ │ └────────────────────┘   │
└───────────────────┴──────────────────────────┘
```

#### 5.6.2 历史记录数据结构

```
{
  id: "history-1",
  name: "订单支付成功趋势试算",        // SQL 首行前22字符
  sql: "SELECT dt, COUNT(1) ...",
  tables: ["dwd_order_detail"],
  executedAt: "2026-03-30 09:41:12",
  status: "success"
}
```

#### 5.6.3 历史记录生成规则

- 每次 SQL 执行成功后自动追加到历史列表头部
- 查询名称取 SQL 第一个非空行的前 22 个字符
- 涉及表列表从当前 `sqlSelectedTables` 或 SQL 中提取的表引用获取
- 最多保留 8 条历史记录

#### 5.6.4 回填行为

- 点击"载入草稿"或"回填 SQL"时：
  1. 如当前有未保存内容，弹出 `confirm` 确认是否覆盖
  2. 将记录中的 SQL 填充到编辑器
  3. 恢复引用表列表
  4. 清空上次执行结果
  5. 关闭弹窗
  6. 弹出通知"草稿已载入"/"历史 SQL 已回填"
- **不会自动重新执行**，用户需手动点击"执行"

### 5.7 多表 SQL 查询与 JOIN

#### 5.7.1 多表选择

- 用户可在左侧面板点击多张表，逐个加入"引用表"列表
- 引用表列表无数量上限
- 引用表栏中的标签可点击切换焦点表，右侧字段面板联动展示对应字段
- 可通过标签上的"×"按钮移除已引用表

#### 5.7.2 SQL 模板生成

当用户选择表时，如果编辑器为空，系统自动生成模板 SQL：

**单表模板：**
```sql
SELECT
  *
FROM wlyd_mc_beijing.dwd_order_detail
LIMIT 100;
```

**多表模板（2张表时）：**
```sql
SELECT
  a.*,
  b.*
FROM wlyd_mc_beijing.dwd_order_detail a
LEFT JOIN wlyd_mc_beijing.ads_user_profile b
  ON a.user_id = b.user_id
LIMIT 100;
```

- 仅当编辑器为空时才自动填充模板
- 编辑器已有内容时不覆盖

#### 5.7.3 JOIN 编写方式

系统不自动组装 JOIN 逻辑，用户需手动编写：
- 通过左侧面板选择表
- 通过右侧面板点击字段插入到 SQL
- 通过"插入表名"按钮快速输入库.表名
- 用户自行编写 ON 条件和其他 SQL 逻辑

## 6. 用户场景与交互流程

### 6.1 从一级菜单进入

**场景**：用户主动打开即席查询工作台进行数据分析。

**流程**：
1. 用户点击顶部导航"即席查询"
2. 进入 Notebook 工作台（notebook.html）
3. 左上方显示 Notebook 列表，左下方显示数据表浏览器
4. 用户可创建新 Notebook 或打开已有 Notebook
5. 在 Notebook 中添加 SQL Cell 编写查询
6. 点击运行按钮或按 Ctrl+Enter 执行

### 6.2 从详情页快捷进入

**场景**：用户在资产详情页查看表信息后，想要立即查询数据。

**流程**：
1. 用户在详情页点击"去查询"按钮
2. 系统调用 `openSQLPage(id)` 跳转到 SQL 工作台（page-sql）
3. 自动执行以下操作：
   a. 将当前表加入引用表列表
   b. 生成模板 SQL（`SELECT * FROM 库.表名 LIMIT 100;`）
   c. 右侧字段面板加载该表字段
   d. 重置筛选为"全部"视图
4. 用户可直接执行或修改 SQL 后执行

### 6.3 无权限拦截

**场景**：用户尝试查询没有权限的资产。

**流程**：
1. 从详情页进入查询时，如果当前表无查询权限
2. 跳转到无权限拦截页（page-sql-noperm）
3. 页面居中显示：
   - 🔒 图标
   - "您暂无该资产的查询权限"
   - "申请通过后即可执行 SQL 查询"
   - [申请数据权限] 按钮
4. 点击按钮打开权限申请弹窗

## 7. 权限校验规则

### 7.1 数据表可见性

- 左侧数据表面板**仅展示**用户已有查询权限的表（`SQL_PERMISSION_TABLES`）
- 无权限的表在列表中不可见

### 7.2 执行前权限校验

SQL 执行前经过三层校验：

| 校验步骤 | 条件 | 失败处理 |
|---------|------|---------|
| 1. 空值校验 | SQL 编辑器内容为空 | 弹出通知"请输入 SQL 语句"，阻止执行 |
| 2. 表引用校验 | SQL 中未引用任何表且未选择任何引用表 | 状态栏显示红色错误"请至少选择一张有权限的数据表，或在 SQL 中引用表名" |
| 3. 权限校验 | SQL 中引用的表不在 `SQL_PERMISSION_TABLES` 中 | 状态栏显示红色错误"当前 SQL 引用了您暂无权限的表：xxx"，并弹出通知提示申请权限 |
| 4. 语法校验 | SQL 存在明显语法问题（如 `SELECT FROM`） | 状态栏显示红色错误"SQL 语法校验失败，请检查关键字、表名或字段名" |

### 7.3 表引用解析规则

使用正则 `/\b(?:from|join)\s+([a-zA-Z0-9_.]+)/ig` 从 SQL 中提取 FROM / JOIN 后的表引用。

权限匹配逻辑（`isAuthorizedSQLTable`）：
- 完整匹配 `fullName`（如 `wlyd_mc_beijing.dwd_order_detail`）
- 匹配 `id`（如 `dwd_order_detail`）
- 匹配 `*.id` 格式（如 `xxx.dwd_order_detail`）

以上三种方式任一匹配即视为有权限。

### 7.4 草稿/历史回填时的权限规则

- 回填草稿或历史查询时，仅恢复 SQL 和引用表列表，不自动执行
- 用户点击"执行"时，系统会重新进行完整的权限校验
- 如果原先有权限的表现在已过期，执行将被阻止

### 7.5 数据可见性

- 草稿和历史查询仅创建者本人可见
- 查询结果仅在线预览，不支持导出和分享

## 8. 异常处理机制

### 8.1 异常类型与处理

| 异常类型 | 触发条件 | 错误信息 | 处理方式 |
|---------|---------|---------|---------|
| SQL 为空 | 编辑器无内容时点击执行 | "请输入 SQL 语句" | 弹出通知，不执行 |
| 未引用表 | SQL 未引用表且未选择引用表 | "请至少选择一张有权限的数据表，或在 SQL 中引用表名" | 状态栏红色提示 |
| 无权限 | SQL 引用了未授权表 | "当前 SQL 引用了您暂无权限的表：{表名}" | 状态栏红色提示 + 通知提示申请权限 |
| SQL 语法错误 | 明显语法错误 | "SQL 语法校验失败，请检查关键字、表名或字段名" | 状态栏红色提示 + 弹出通知 |
| 执行失败 | 后端执行异常 | "查询执行失败，请稍后重试" | 状态栏红色提示 |
| 保存草稿失败 | SQL 为空时保存 | "请先输入 SQL 再保存草稿" | 弹出通知，不保存 |
| 格式化失败 | SQL 为空时格式化 | "请输入 SQL 语句" | 弹出通知 |

### 8.2 处理原则

- **不清空编辑器**：任何错误都保留当前 SQL 内容
- **不清空结果**：语法错误和权限错误隐藏结果区，但 SQL 保持不变
- **未保存提醒**：回填草稿/历史时，如有未保存变更，弹出 confirm 确认

## 10. Notebook 工作台

Notebook 工作台是即席查询的另一种形态，采用 Jupyter 风格的多 Cell 渐进式数据探索模式，作为独立页面 `notebook.html` 存在。

### 10.1 整体布局

```
┌──────────────────────────────────────────────┐
│ Header（深色导航栏）                            │
├──────────────┬───────────────────────────────┤
│ 左侧面板      │ 工具栏（标题 / 保存 / 分享 / 运行）│
│ 260px        ├───────────────────────────────┤
│              │                               │
│ Notebook列表  │  Cell 编辑区                   │
│ ────────────  │ （SQL / Markdown / Chart）     │
│ 数据表浏览器   │                               │
│              │                               │
├──────────────┴───────────────────────────────┤
│ 状态栏（连接状态 / 保存时间 / Cell数量）          │
└──────────────────────────────────────────────┘
```

### 10.2 左侧面板

分为上下两个可拖拽调节大小的区域：

**上半区 — Notebook 列表**
- "我的" Notebook 列表：按最近修改排序，显示标题、Cell 数量、时间
- "收到的分享" 列表：显示分享者信息，只读标记
- 搜索框：模糊匹配标题
- "+" 按钮：新建 Notebook

**下半区 — 数据表浏览器**
- 搜索框："搜索表名或描述..."
- 表列表：两行布局，第一行技术表名（等宽字体），第二行 schema + 中文描述
- 点击表名展开字段列表（含字段名 + 类型）
- 点击字段插入到当前焦点 SQL Cell 的编辑器

### 10.3 Cell 类型

#### SQL Cell
- **编辑器**：Monaco Editor，SQL 语法高亮，表/字段自动补全
- **自动高度**：最小 60px，最大 400px，随内容自适应
- **头部**：拖拽手柄、类型标签（蓝色 SQL）、序号、操作按钮（运行/上移/下移/删除）
- **快捷键**：Ctrl+Enter 执行当前 Cell，Shift+Enter 执行并跳到下一个
- **执行结果**：Cell 下方内联显示 — 状态行（✓ N行 · 耗时）+ 数据表格
- **结果操作**：导出 CSV、转为图表

#### Markdown Cell
- 默认渲染模式（marked.js 解析）
- 双击进入编辑模式（textarea）
- 点击外部或 Esc 退出编辑，自动渲染
- 空 Cell 显示占位符"双击编辑 Markdown..."

#### Chart Cell
- 绑定某个 SQL Cell 的执行结果作为数据源
- 图表类型切换：柱状图 / 折线图 / 饼图 / 散点图
- 设置面板：数据源选择、X 轴字段、Y 轴字段
- 使用 ECharts 渲染
- 源 SQL 重新执行时图表自动刷新

### 10.4 Notebook 管理

| 操作 | 说明 |
|------|------|
| 新建 | 默认标题"未命名 Notebook"，预置一个空 SQL Cell |
| 自动保存 | 内容变更后 3 秒触发自动保存（localStorage） |
| 手动保存 | Ctrl+S，弹出 toast 通知 |
| 删除 | 确认对话框后删除 |
| 标题编辑 | 工具栏中可直接编辑 Notebook 标题 |

### 10.5 分享功能

- 点击"分享"按钮打开模态框
- 生成只读链接，支持设置有效期（7天 / 30天 / 永久）
- 接收方进入只读模式：
  - 顶部蓝色横幅"🔒 只读模式 · 来自 XXX 的分享"
  - Cell 不可编辑，运行按钮隐藏
  - 工具栏仅显示"复制到我的 Notebook"按钮
- "复制到我的 Notebook"：Fork 一份副本，标题加 " (副本)" 后缀

### 10.6 全部运行

- 按顺序从上到下执行所有 SQL Cell
- 某个 Cell 执行失败时暂停，弹出确认"Cell #N 执行失败，是否继续执行后续 Cell？"
- 全部完成后弹出 toast"全部运行完成"

### 10.7 Cell 管理交互

- **添加 Cell**：每个 Cell 之间有隐藏的"+" 按钮（hover 显示），下拉选择 SQL 或 Markdown
- **拖拽排序**：通过 Cell 左侧拖拽手柄（⠿）实现
- **删除**：至少保留一个 Cell，否则提示"至少保留一个 Cell"
- **焦点管理**：点击 Cell 切换焦点，焦点 Cell 左侧显示蓝色指示条

## 11. 数据结构定义

### 11.1 权限表（SQL_PERMISSION_TABLES）

```typescript
interface PermissionTable {
  id: string;            // 表标识，如 "dwd_order_detail"
  cnName: string;        // 中文名，如 "订单明细宽表"
  fullName: string;      // 库.表名，如 "wlyd_mc_beijing.dwd_order_detail"
  sourceType: string;    // 来源类型，如 "数仓引擎"、"业务库"
  sourceSystem: string;  // 来源系统，如 "MaxCompute"、"SelectDB"
  type: string;          // 资源类型，如 "table"
  favorite: boolean;     // 是否收藏
  recent: boolean;       // 是否最近使用
}
```

### 11.2 表字段（SQL_TABLE_FIELDS）

```typescript
interface TableField {
  name: string;     // 字段名，如 "order_id"
  type: string;     // 数据类型，如 "STRING"、"DECIMAL(18,2)"
  comment: string;  // 字段注释，如 "订单唯一标识"
}

// 结构：Record<tableId, TableField[]>
```

### 11.3 SQL 草稿（MOCK_SQL_DRAFTS）

```typescript
interface SQLDraft {
  id: string;         // 草稿ID
  name: string;       // 草稿名称
  sql: string;        // SQL 文本
  tables: string[];   // 引用表ID列表
  updatedAt: string;  // 最后保存时间
}
```

### 11.4 查询历史（MOCK_SQL_HISTORY）

```typescript
interface SQLHistory {
  id: string;          // 记录ID
  name: string;        // 查询摘要（SQL首行前22字符）
  sql: string;         // SQL 文本
  tables: string[];    // 涉及表ID列表
  executedAt: string;  // 执行时间
  status: string;      // 执行状态 "success" | "error"
}
```

### 11.5 Notebook 数据结构

```typescript
interface Notebook {
  id: string;
  title: string;
  createdAt: string;    // ISO 时间戳
  updatedAt: string;    // ISO 时间戳
  sharedBy?: string;    // 分享者（仅收到的分享）
  cells: Cell[];
}

interface Cell {
  id: string;
  type: "sql" | "markdown" | "chart";
  source: string;        // 源代码（SQL/Markdown）
  output: any;           // 执行结果（SQL Cell）
  status?: string;       // 执行状态（SQL Cell）
  sourceCell?: string;   // 数据源 Cell ID（Chart Cell）
  chartType?: string;    // 图表类型（Chart Cell）
  xField?: string;       // X轴字段（Chart Cell）
  yField?: string;       // Y轴字段（Chart Cell）
}
```

### 11.6 Notebook 数据表

```typescript
interface NotebookTable {
  id: string;            // 表标识
  schema: string;        // 库名，如 "wlyd_mc"
  name: string;          // 表名，如 "dwd_user_basic_info_d1"
  comment: string;       // 中文描述
  fields: {
    name: string;
    type: string;
    comment: string;
  }[];
}
```
