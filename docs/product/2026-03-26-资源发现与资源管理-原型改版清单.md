# 资源发现与资源管理 - 原型改版清单

## 1. 改版目标

基于当前产品边界调整，对现有原型进行改版：

- 将“资源目录”改为“资源发现”
- 将“资源发现”聚焦为统一查询入口
- 将“资源管理”聚焦为治理后台
- 让资源与资产在查询侧统一，在治理侧分状态管理

本清单聚焦：

- 原型页面改什么
- 哪些模块该删
- 哪些模块该新增
- 每个页面建议怎么排版

**关联模块 PRD：** 改版项落地后的**功能需求与验收**以 `docs/product/2026-04-04-*-模块PRD.md` 及总册 `2026-03-28-数据资产管理平台-资源发现与详情页治理PRD.md` 为准；本清单侧重**原型改动台账**，与模块 PRD 冲突时以模块 PRD 及变更说明为准。

| 改版主域 | 对应模块 PRD（示例） |
|----------|----------------------|
| 资源发现 / 列表与筛选 | `2026-04-04-资源发现-模块PRD.md` |
| 资源管理 / 治理后台 | `2026-04-04-资源管理-模块PRD.md` |
| 统一详情 / Tab | `2026-04-04-统一详情页-模块PRD.md` |

## 2. 一级导航改版

### 2.1 导航名称调整

当前：

- 资产检索
- 资产目录
- 资源目录
- 资源管理
- 血缘追溯

建议改为：

- 数据搜索
- 资产目录
- 资源发现
- 资源管理
- 血缘追溯

### 2.2 调整原因

- `资产检索` 更像页面功能名，建议升级为更统一的入口名 `数据搜索`
- `资源目录` 改为 `资源发现`，更符合用户查询心智
- `资源管理` 继续保留，用于承接治理后台

## 3. 页面级改版清单

## 3.1 数据搜索页

### 当前问题

- 搜索页主要输出资产结果
- 当资产未命中时，才降级展示资源
- 这不符合“资产和资源一起查出来”的目标

### 改版建议

- 搜索结果页改成统一结果页
- 顶部增加结果切换：
  - 全部
  - 仅资产
  - 仅资源
- 每条结果明确展示：
  - 类型
  - 当前状态
  - 是否为资产

### 建议新增模块

- 搜索结果分组展示
  - 优先推荐资产
  - 同时展示相关资源
- 筛选项
  - 类型
  - 来源平台
  - 标签
  - 更新时间
  - 资产/资源
- 推荐区
  - 热门资产
  - 热门资源
  - 最近新增资源

### 原型改造优先级

- P0：统一结果页
- P0：增加“全部 / 仅资产 / 仅资源”
- P1：增加按状态筛选
- P1：增加推荐区

---

## 3.2 资产目录页

### 页面定位

- 面向业务用户按目录查找已上架资产
- 不承接资源治理

### 建议保持不变的部分

- 业务目录树
- 资产卡片式浏览
- 类型筛选
- 标签筛选

### 建议优化的部分

- 增加“来源资源”信息
  - 让用户知道该资产来自哪个资源
- 增加“最近更新 / 使用热度 / 收藏量”
- 增加“关联资源推荐”

### 原型改造优先级

- P1：在资产详情页里增加来源资源信息

---

## 3.3 资源发现页

### 页面定位

- 面向普通用户统一查找资源与资产
- 重点帮助发现未资产化资源
- 不承接治理动作

### 当前问题

- 当前资源目录带有明显治理属性
- 页面上存在上架状态筛选和“上架”动作
- 容易与资源管理职责冲突

### 当前建议移除的内容

- 状态快捷筛选中的：
  - 待审核
  - 驳回待修改
  - 已上架
  - 不上架
- 列表操作中的：
  - 上架
- 页面中的治理导向文案

### 当前建议保留的内容

- 搜索资源名称
- 平台/来源展示
- 类型筛选
- “我的数据”
- “未分配目录”可转移到治理后台，不建议继续放这里

### 建议新增的内容

- 结果类型切换
  - 全部
  - 仅资产
  - 仅资源
- 资源状态简化展示
  - 资产
  - 非资产
  - 不强调治理过程状态
- 搜索维度扩展
  - 中文名
  - 英文名
  - 别名
  - 标签
  - 来源系统
  - 负责人
- 推荐区
  - 最近新增资源
  - 热门资源
  - 同域相关资源
- 详情页动作
  - 查看详情
  - 收藏
  - 复制名称
  - 申请权限
  - 建议资产化（后续可选）

### 页面结构建议

- 顶部
  - 大搜索框
  - 快速筛选条：全部 / 仅资产 / 仅资源
- 左侧
  - 来源系统树 或 业务域树
  - 标签筛选
  - 类型筛选
- 中间
  - 统一结果列表
- 右侧
  - 推荐资源
  - 最近浏览
  - 收藏夹

### 列表字段建议

- 名称
- 中文名
- 类型
- 是否资产
- 来源平台 / 来源系统
- 负责人
- 更新时间
- 标签
- 可申请状态

### 原型改造优先级

- P0：页面标题改为“资源发现”
- P0：去掉列表中的“上架”按钮
- P0：去掉治理型状态筛选
- P0：增加“全部 / 仅资产 / 仅资源”
- P1：增加推荐区
- P1：增加标签筛选
- P1：增加统一详情跳转

## 3.4 详情页

### 页面定位

- 资源与资产共用同一套详情页
- 根据当前状态展示不同动作

### 当前问题

- 详情页里仍然混有“发起上架申请”等治理动作
- 用户详情页和治理详情页边界不清

### 建议改造方向

- 普通详情页只展示用户可见动作
- 治理动作放到资源管理的详情页或治理抽屉中

### 建议动作分层

#### 用户视角详情动作

- 查看字段
- 查看血缘
- 收藏
- 复制名称
- 申请权限

#### 治理视角详情动作

- 补充信息
- 编辑标签
- 调整目录归属
- 提交上架
- 标记不上架
- 下架 / 恢复

### 建议实现方式

- 共用详情内容区
- 根据入口来源决定操作按钮
  - 从“资源发现”进入：只显示用户动作
  - 从“资源管理”进入：显示治理动作

### 原型改造优先级

- P0：将治理动作从“资源发现”详情入口里弱化
- P1：详情页支持根据入口切换按钮区

---

## 3.5 资源管理页

### 页面定位

- 面向数仓工程师的治理后台

### 当前问题

- 当前只拆成：
  - 待审核上架申请
  - 待维护资源
  - 异常/待处理
- 不足以承接你现在定义的完整治理流程

### 建议改造成 5 个模块

#### 模块 1：工作台

- 今日待办
- 待审核数量
- 待维护数量
- 驳回待处理数量
- 已下架数量
- 异常数量

#### 模块 2：上下架管理

- 待维护
- 待审核
- 已驳回
- 已上架
- 不上架
- 已下架

#### 模块 3：标签管理

- 标签字典
- 待补标签资源
- 批量打标
- 系统标签校验

#### 模块 4：目录归属管理

- 未归属目录
- 待调整目录
- 批量调整归属

#### 模块 5：异常处理

- 元数据异常
- 字段变更异常
- 权限异常
- 血缘异常

### 原型页面结构建议

- 顶部统计卡片
  - 我的待办
  - 待审核
  - 待维护
  - 已驳回
  - 异常
- 主体区 tab
  - 工作台
  - 上下架管理
  - 标签管理
  - 目录归属
  - 异常处理
  - 操作记录

### 原型改造优先级

- P0：将当前“资源管理工作台”改造成多 tab 后台
- P0：新增“上下架管理”主 tab
- P0：新增“标签管理”
- P0：新增“目录归属管理”
- P1：新增“操作记录”

## 6. 需要你后续拍板的改版决策

- `不上架` 是否需要审核
- `建议资产化` 是否在资源详情页开放
- 目录归属是否先只支持单目录
- 目录体系是否按“业务域 / 主题域 / 数据对象”重构
