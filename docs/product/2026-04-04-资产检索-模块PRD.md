## 2. 模块定位与背景

### 2.1 一句话定位

**资产检索**是面向消费侧的**统一关键词入口**：用户通过主搜索框输入关键字，先获得**联想下拉**快速命中，再可执行**全文搜索结果列表**，最终进入**统一详情页**理解与申请权限。

### 2.2 要解决的问题

- 用户已知**表名/中文名/别名/负责人/标签**等片段时，需要比「进资源发现再筛」更短的路径。
- 需要**同时覆盖已上架资产与资源侧对象**，与「仅目录浏览」形成互补。
- 在输入过程中给出**可操作的候选**，减少「搜完才知道有没有」的试错成本。

### 2.3 非目标（本模块不做）

- 不上架、改目录、批量治理、审批流配置（在**资源管理**或其它后台完成）。
- 不替代**资源发现**的多维筛选与物理树导航；检索无结果时的引导可链到资源发现，但不在本 PRD 展开其筛选细则。
- 原型中「AI 助手」等占位能力，**不纳入本期功能验收**（可保留入口与文案，以后续 PRD 为准）。

## 3. 目标与成功标准

### 3.1 产品目标

- 用户**输入即有反馈**：有关键字时展示联想面板；无关键字时收起联想，主区可展示发现辅助内容（与原型一致）。
- **键盘可操作**：方向键选择候选，Enter 进入详情或触发全文搜索，Esc 收起下拉。
- 进入详情时携带正确的 **`domain`（asset | resource）与对象 id**，与统一详情页模块要求一致。

### 3.2 成功指标（可后续量化）

- 联想点击率、联想→详情转化率、绕过联想直接全文搜索占比。
- 检索无结果后的「调整关键词 / 跳转资源发现」使用率（若埋点）。

## 4. 用户与场景

### 4.1 主要用户

| 角色 | 典型诉求 |
|------|----------|
| 分析师 / 取数用户 | 凭印象搜表名、指标名、标签，快速打开详情申请权限 |
| 数据开发 | 确认某对象在资产还是资源、负责人与来源系统 |
| 新用户 | 通过热门词、专题卡片降低冷启动成本（原型已展示） |

### 4.2 典型场景（用户故事摘要）

1. **输入几个字母**：联想列出最相关的若干资产/资源，方向键选中后 Enter 进详情。
2. **无快捷命中**：下拉内提示可 Enter 全文搜索；用户按 Enter 后查看完整结果列表并筛选范围。
3. **只想看资源或只看资产**：在全文结果区切换「全部 / 仅资产 / 仅资源」。
4. **从热门词起步**：点击热门标签填充关键字并触发搜索（与原型 `fillSearch` 行为一致）。

## 6. 功能需求

### 6.1 主搜索框（FR-SRC-01）

- **占位提示**：与原型一致，如「搜索资产名称、描述、负责人、标签…」（实现时可随检索字段扩展微调文案）。
- **浏览器行为**：关闭浏览器自带自动完成（`autocomplete="off"` 或等价策略），避免与平台联想层冲突。
- **清空**：输入清空时收起联想，并恢复默认发现区展示（与原型 `resetSearchResultState` 语义一致）。
- **验收**：焦点在框内时行为符合 §6.2～§6.5；清空后无残留下拉与错误结果区。

### 6.2 联想下拉 — 显示与关闭（FR-SRC-02）

- **打开条件**：输入框内容经 `trim` 后**非空**时展示联想面板；**空串**时不展示。
- **再次打开**：输入框 **focus** 且当前有关键字时，重新计算并展示联想（与原型 `handleMainSearchFocus` 一致）。
- **关闭**：`Esc` 收起；选中一条进入详情后收起；点击「搜索」执行全文搜索前收起。
- **实现建议**：对接服务端时应对输入做 **debounce**（如 200～400ms），避免每个字符都打满请求；当前原型为本地 Mock，可无 debounce。
- **验收**：无闪烁错位（面板贴紧搜索框）、不与页面滚动条严重遮挡（z-index 与最大高度与原型 `search-suggest-list` 一致或可配置）。

### 6.3 联想 — 数据源与条数（FR-SRC-03）

- **范围**：同时检索**资产**与**资源**两路数据，合并展示。
- **条数上限**：默认最多 **8** 条候选（与原型 `slice(0, 8)` 一致）；产品可在技术方案中按性能调整为 10，但需统一端上展示规则。
- **排序**：按**相关度评分**降序（原型为 `buildSearchSuggestionScore`：名称、中文名、别名、说明、标签、来源等加权）；服务端实现时需保证**可解释的稳定排序**（同分按类型或更新时间 tie-break，文档化即可）。
- **验收**：同一关键字多次请求（Mock 或接口）结果顺序稳定；资产/资源均可能被命中。

### 6.4 联想 — 单条展示与命中说明（FR-SRC-04）

- **展示要素**：类型图标、**名称**（技术名为主）、**资产/资源**域标签、**对象类型**标签（表/视图/指标等）、副文（中文名或摘要/来源简述）、**命中维度**标签（如「命中中文名」「命中别名」）、负责人、右侧操作提示。
- **高亮**：关键字在名称、副文、命中标签中 **HTML 高亮**（与原型 `highlight` 一致）；注意 XSS 安全，服务端返回需转义或由前端安全拼接。
- **头部文案**：展示「已命中 N 条候选」；无结果时副文案提示可 Enter 全文搜索（与原型 `search-suggest-header-desc` 语义一致）。
- **验收**：命中维度与用户输入有对应关系，不误导；高亮样式可读（如 `<em>` 配色与原型一致）。

### 6.5 联想 — 键盘与鼠标（FR-SRC-05）

- **↓ / ↑**：在存在候选时切换**选中项**，选中态样式与原型 `active` 一致；无候选时不拦截（或仅不移动选中）。
- **Enter**：若存在**当前选中项**，进入 §6.6；否则触发 **全文搜索**（与原型 `handleMainSearchKeydown` 一致）。
- **Esc**：收起联想面板，不清空输入。
- **鼠标**：`mouseenter` 项时同步选中索引；点击项进入 §6.6。
- **验收**：键盘与鼠标切换选中不打架；Enter 行为与按钮「搜索」在「无选中项」时等价。

### 6.6 联想 — 选中后跳转（FR-SRC-06）

- **行为**：将输入框文案更新为候选的展示名（`display` 优先，与原型一致），收起联想，调用 **`openDetail(domain, id)`**。
- **domain**：资产为 `asset`，资源为 `resource`（与统一详情页 FR-DTL-01 一致）。
- **验收**：任意候选进入详情后，详情页数据与列表/卡片同源字段一致，返回路径合理（见详情页「返回」规则）。

### 6.7 全文搜索（FR-SRC-07）

- **触发**：点击「搜索」按钮，或在联想无选中项时按 **Enter**。
- **视图切换**：隐藏默认发现面板，展示搜索结果区（`search-result-view`），并渲染统一检索结果（与原型 `doSearch` / `renderUnifiedSearchResults` 一致）。
- **匹配字段**：至少覆盖名称、展示名/摘要、标签、负责人、来源/平台等（与 `getUnifiedDiscoveryRecords` 语义对齐；具体字段以实现/API 为准）。
- **验收**：关键字变化后结果与「共 N 条」一致；结果行点击同样 `openDetail`。

### 6.8 全文结果 — 筛选与排序（FR-SRC-08）

- **范围筛选**：**全部 / 仅资产 / 仅资源** 切换后结果集与计数立即更新（与原型 `setSearchResultScope` 一致）。
- **排序**：原型提供下拉（使用热度、创建/更新时间升降序）；若一期不接后端排序，需**下线控件或标注占位**，避免假交互。
- **空状态**：无结果时主列表空态 + **fallback 区**提示可前往资源发现（与原型 `fallback-section` 一致）。
- **验收**：筛选与列表、统计数字三方一致；空态文案清晰。

### 6.9 热门词与辅助发现（FR-SRC-09）

- **热门词**：点击后将关键字写入搜索框并触发联想更新 + **全文搜索**（与原型 `fillSearch` 一致）。
- **发现 Tab / 专题**：用于无搜索时的浏览引导；与检索核心链路解耦，**不阻塞** §6.1～§6.8 验收。

## 7. 权限与安全

- 检索结果应遵守**行级/对象级可见性**（用户仅能看见有权知晓的对象）；具体规则与总册及权限模块一致。
- 高亮与摘要展示需注意**敏感字段脱敏**（与详情页样例脱敏策略衔接）。

## 8. 验收标准（模块级检查清单）

- [ ] 非空关键字出现联想面板，空关键字收起，焦点重现时行为正确。
- [ ] 联想最多 8 条（或经书面确认的调整值），资产/资源标签、类型标签、命中说明、高亮正确。
- [ ] ↑↓ / Enter / Esc / 鼠标悬停与点击行为与 §6.5、§6.6 一致。
- [ ] 全文搜索切换结果视图，统计与列表一致；范围筛选有效；空态与 fallback 展示正确。
- [ ] 任意命中路径进入详情时 **`domain` + `id`** 正确，详情页可正常加载。
- [ ] 对接真实接口时具备 **debounce** 与基础错误提示（超时、服务不可用）；Mock 阶段可标注技术债。
