## 2. 模块定位与背景

### 2.1 一句话定位

**即席查询**是面向已授权数据的**轻量 SQL Notebook 工作台**：用户在「我有权限的表」范围内编写和组织 SQL、说明文本与图表内容，**执行前校验引用表权限**，在线预览结果，支持**转为图表**与**分享只读链接**（有效期可选 7天/30天/永久）。

### 2.2 要解决的问题

- 「找数 → 看详情 → 申请权限」之后，缺少**统一入口**做跨表探查与简单 JOIN 分析。
- 需避免用户写出引用**无权限表**的 SQL 却在执行后才失败；应**前置拦截**并引导申请权限。

### 2.3 非目标（本模块不做）

- 替代专业 BI 工具的复杂图表与报表能力（支持基础图表转换，但复杂报表不在范围内）。
- 专业 IDE 能力：多文件工程、Git、复杂调试器、AI 自动生成完整业务 SQL（辅助插入表名/字段名可保留）。
- 替代调度开发、数据生产任务提交。

---

## 3. 目标与成功标准

### 3.1 产品目标

- 用户从**一级菜单**或从**详情页**进入时，能快速进入 Notebook 工作台并开始组织分析内容。
- **执行**前对 SQL 中引用的表做权限校验；不通过时明确列出无权限对象并阻止返回结果。
- 当前分析内容可保存为 Notebook，并在后续继续编辑、补充与复用。

### 3.2 成功指标（可后续量化）

- 工作台活跃用户数、人均每周执行次数、从无权限拦截到权限申请成功的转化率。

## 4. 用户与场景

### 4.1 主要用户

| 角色 | 典型场景 |
|------|----------|
| 业务分析师 | 多表关联做临时分析 |
| 数据开发 | 快速验证表数据、字段名 |
| 业务用户 | 从详情「去查询」单表模板起步 |

### 4.2 典型场景（用户故事摘要）

1. **全局新建**：点击「新建 Notebook」，创建空白分析页，逐个新增 SQL / Markdown Cell 开始分析。
2. **从详情带入**：从详情「去查询」进入 Notebook，预置查询上下文或首个 SQL Cell。
3. **协作分享**：分析完成后生成只读链接，供他人查看结果并复制到自己的 Notebook。
4. **持续分析**：保存当前 Notebook，在左侧列表中继续编辑；支持一次执行全部 Cell。

## 6. 功能需求

### 6.1 入口与导航（FR-SQL-01）

- **一级菜单**：进入 Notebook 工作台，默认打开当前用户最近一次编辑的 Notebook，或展示欢迎空态。
- **新建 Notebook**：创建新的分析文档，默认生成一个空白 SQL Cell，标题可直接编辑。
- **详情页**：从详情「去查询」进入时，可带入默认分析上下文，帮助用户从单表分析起步。
- **欢迎空态**：当用户尚未打开任何 Notebook 时，页面展示欢迎说明与“新建 Notebook”主按钮。
- **验收**：用户可清晰区分“当前分析文档”“我的 Notebook”“收到的分享”。

### 6.2 数据表区与字段辅助（FR-SQL-02）

- **数据源**：仅展示当前用户具备查询权限的数据表。
- **搜索**：支持按表名或描述搜索。
- **展开查看**：支持展开表节点查看字段清单，帮助用户理解可用字段。
- **辅助编写**：用户可通过点击表名、字段名，辅助完成 SQL 编写。
- **验收**：无权限表不出现在数据表区；表与字段层级清晰可读。

### 6.3 Cell 化分析编辑（FR-SQL-03）

- **Cell 类型**：支持 SQL Cell、Markdown Cell、图表 Cell。
- **新增方式**：用户可在任意 Cell 前后通过“+ 添加”插入新的 SQL 或 Markdown Cell。
- **SQL Cell**：用于编写与执行查询，结果展示在当前 Cell 下方。
- **Markdown Cell**：用于补充分析说明、结论、备注等内容，支持双击进入编辑态。
- **图表 Cell**：可基于 SQL 结果转为可视化图表，用于表达分析结果。
- **顺序组织**：Cell 支持调整顺序与删除，用于整理完整分析过程。
- **验收**：用户可在同一 Notebook 内混合组织查询、说明与图表，形成完整分析过程。

### 6.4 执行、结果与错误反馈（FR-SQL-04）

- **单 Cell 执行**：用户可单独执行某个 SQL Cell，也可触发“全部运行”顺序执行所有可执行 Cell。
- **执行反馈**：执行成功后，在当前 Cell 下方展示结果行数、耗时、结果表格。
- **错误反馈**：执行失败时，错误信息直接展示在当前 Cell 的结果区内，不通过 toast 承载核心报错。
- **中断选择**：全部运行过程中若某个 Cell 失败，可提示用户是否继续执行后续 Cell。
- **自动保存**：用户编辑标题、Cell 内容或结构后，系统自动保存当前 Notebook，减少分析过程中的内容丢失。
- **验收**：用户无需离开当前分析上下文，即可看清每个 Cell 的执行结果与报错原因。

### 6.5 保存、列表与继续分析（FR-SQL-05）

- **保存**：用户可手动保存当前 Notebook；编辑过程中也支持自动保存。
- **Notebook 列表**：左侧展示“我的” Notebook，支持搜索与切换，并显示最近更新时间与 Cell 数量。
- **收到的分享**：分享给当前用户的 Notebook 以独立分组展示，并显示分享来源。
- **继续分析**：用户重新进入后，可从上次 Notebook 继续编辑与运行。
- **删除**：支持删除 Notebook，删除前需二次确认。
- **验收**：Notebook 作为分析载体被持续复用，而非一次性 SQL 页面。

### 6.6 分享与协作（FR-SQL-06）

- **分享方式**：通过顶部「分享」按钮生成 Notebook 只读链接。
- **有效期**：支持设置 7天 / 30天 / 永久。
- **只读协作**：被分享者默认以只读方式查看，不可直接修改原内容，可复制到自己的 Notebook 后继续编辑。
- **只读限制**：只读态下标题编辑、Cell 编辑、保存等修改性操作均不可用。
- **结果复用**：SQL 查询结果可转为图表 Cell，用于沉淀分析结论。
- **图表配置**：图表 Cell 支持选择来源 SQL Cell、图表类型，以及 X/Y 轴字段。
- **范围说明**：本期不支持原始数据导出，不提供 CSV 下载能力。

## 7. 权限、安全与审计

- 列表、解析、执行三处权限判断需**同源**，避免仅前端校验。
- 敏感环境应记录：谁、何时、执行了哪段 SQL（摘要）、涉及表、是否拒绝（供审计）。

## 8. 验收标准（模块级检查清单）

- [ ] 可新建 Notebook，默认生成首个 SQL Cell，并在左侧列表中正常展示与切换。
- [ ] 左侧 Notebook 列表可区分“我的”与“收到的分享”，并支持搜索。
- [ ] 数据表区仅展示有权限的数据表，支持搜索、展开字段、辅助编写 SQL。
- [ ] SQL Cell 可单独执行，结果与错误均展示在当前 Cell 下方。
- [ ] “全部运行”可顺序执行多个 Cell，失败时支持继续/停止决策。
- [ ] Markdown / SQL / 图表 Cell 可混合组织为完整分析内容，且支持新增、排序、删除。
- [ ] 编辑过程中支持手动保存与自动保存，状态栏可反馈当前状态。
- [ ] 「分享」按钮可生成只读链接，并正确展示有效期选项（7天/30天/永久）。
- [ ] 被分享者进入后默认只读，不可直接修改原内容，但可复制到自己的 Notebook。
- [ ] 图表 Cell 可选择数据源 SQL Cell、图表类型与坐标字段。
- [ ] 本期无原始数据导出入口。
