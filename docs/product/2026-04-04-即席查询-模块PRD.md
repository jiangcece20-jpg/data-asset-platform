## 2. 模块定位与背景

### 2.1 一句话定位

**即席查询**是面向已授权数据的**轻量 SQL 工作台**：用户在「我有权限的表」范围内选表、借助字段面板拼装或手写 SQL，**执行前校验引用表权限**，在线预览结果（**不支持导出与分享**）。

### 2.2 要解决的问题

- 「找数 → 看详情 → 申请权限」之后，缺少**统一入口**做跨表探查与简单 JOIN 分析。
- 需避免用户写出引用**无权限表**的 SQL 却在执行后才失败；应**前置拦截**并引导申请权限。

### 2.3 非目标（本模块不做）

- 查询结果**导出、下载、分享链接**（与原型文案一致）。
- 专业 IDE 能力：多文件工程、Git、复杂调试器、AI 自动生成完整业务 SQL（辅助插入表名/字段名可保留）。
- 替代调度开发、数据生产任务提交。

## 3. 目标与成功标准

### 3.1 产品目标

- 用户从**一级菜单「新建查询」**或从**详情页**进入时，能在左侧看清**当前账号有查询权限的表**并完成选表。
- **执行**前对 SQL 中 `FROM`/`JOIN` 引用的表做权限校验；不通过时明确列出无权限对象并引导申请（`page-sql-noperm` / `openPermModal`）。
- 草稿与历史可查可载入（与原型 Mock 行为一致；真实持久化由接口定义）。

### 3.2 成功指标（可后续量化）

- 工作台活跃用户数、人均每周执行次数、从无权限拦截到权限申请成功的转化率。

## 4. 用户与场景

### 4.1 主要用户

| 角色 | 典型场景 |
|------|----------|
| 业务分析师 | 多表关联做临时分析 |
| 数据开发 | 快速验证表数据、字段名 |
| 业务用户 | 从详情「去查询」单表模板起步 |

### 4.2 典型场景（用户故事摘要）

1. **全局新建**：点击「新建查询」，清空上下文，从左侧选表再写 SQL。
2. **从详情带入**：`openSQLPage(id)` 预选表并预填模板 SQL（与原型一致）。
3. **无权限**：直接打开某表查询页时若无权，进入拦截态并 **申请数据权限**。
4. **保存草稿 / 历史**：保存当前 SQL 与引用表；从历史记录恢复执行。

## 5. 信息架构与页面结构

### 5.1 主工作台（`page-sql`）

| 区域 | 说明 |
|------|------|
| **上下文栏**（`sql-context-bar`） | 返回、标题与说明、上下文标签（如「全局查询工作台」/单表上下文）、合规提示、「仅展示已有权限表」、新建查询 / 保存草稿 / 历史查询 |
| **左栏** | 我有权限的表：概览数字、搜索、筛选（全部 / 最近使用 / 收藏常用）、表列表（`renderSQLTableList`） |
| **中栏** | 当前引用表标签、空状态引导、SQL 编辑器、执行 / 格式化 / 保存草稿、状态条、结果表与结果说明 |
| **右栏** | 字段辅助：按当前聚焦表展示字段，搜索，插入表名 / 插入字段 |

### 5.2 无权限页（`page-sql-noperm`）

- 文案说明暂无查询权限，主按钮跳转权限申请（与原型一致）。

### 5.3 弹窗

- **保存草稿**（`modal-sql-draft`）：草稿名、引用表展示、SQL 预览、保存。
- **历史查询与草稿**（`modal-sql-history`）：草稿列表与历史列表分栏，支持载入（与原型 `renderSQLDraftList` / `renderSQLHistoryList` 语义一致）。

### 5.4 与相邻模块的关系

| 模块 | 关系 |
|------|------|
| 统一详情页 | 「去查询」打开工作台并带默认表/SQL 模板。 |
| 权限中心 / 申请 | 无权限时引导申请；通过后表出现在左侧列表。 |
| 我的 | 可与收藏、最近使用数据源对齐（左侧「收藏常用」等）。 |

## 6. 功能需求

### 6.1 入口与导航（FR-SQL-01）

- **一级菜单**：进入工作台；「新建查询」重置选中表、清空编辑器、隐藏上次结果（`openSQLWorkbenchFromMenu`）。
- **详情页**：传入资源 id，校验权限后预选表并预填 SQL（`openSQLPage`）；无权则走无权限页或 toast 策略（与产品统一）。
- **返回**：`goBack` 与全局页历史一致。
- **验收**：上下文标签（`sql-context-tag`）能反映单表/全局场景（若产品需要）。

### 6.2 授权表列表（FR-SQL-02）

- **数据源**：仅展示当前用户**具备查询权限**的表（及扩展：视图等若策略允许）。
- **筛选**：全部、最近使用、收藏常用；搜索匹配中文名、英文名、库表名（占位实现可渐进增强）。
- **多选**：支持多张表进入「当前引用表」，点击标签切换右栏字段聚焦（`sqlFocusedTableId`）。
- **验收**：列表条数与权限服务一致；无权限表不出现在左侧。

### 6.3 SQL 编辑与辅助（FR-SQL-03）

- **编辑器**：多行文本；占位提示与原型一致。
- **插入**：从右栏插入表全名、字段名到光标处（`insertIntoSQLEditor`）。
- **格式化**：基础关键字换行与大写（与 `formatSQL` 行为一致；可替换为更强格式化服务）。
- **清空**：清空编辑器与结果区，更新未保存状态。
- **验收**：大脚本下输入仍可用（性能基线由技术方案定）。

### 6.4 执行与权限校验（FR-SQL-04）

- **触发**：执行按钮；空 SQL 报错。
- **表引用解析**：从 SQL 中解析 `FROM`/`JOIN` 后对象名（与 `extractSQLTableReferences` 语义一致；实现可加强引号、子查询边界）。
- **校验**：任一引用不在授权列表则**不执行**，状态条列出无权限表名，并可 `showNotification`；结果区隐藏。
- **语法校验**：产品可定义轻量规则或对接引擎解析错误（原型含简单反例拦截）。
- **执行成功**：展示行数与耗时（真实系统以引擎为准）、结果表格；**明确仅预览**、默认截断行数（原型「前 8 行预览」为演示）。
- **验收**：越权 SQL **绝不能**返回真实数据；审计日志由安全方案覆盖。

### 6.5 草稿与历史（FR-SQL-05）

- **保存草稿**：名称必填或默认名；保存 SQL 文本与引用表 id 列表；支持更新已有草稿（`sqlCurrentDraftId`）。
- **历史查询**：每次成功执行写入历史（条数上限与原型一致或可配置）；列表展示摘要、时间、状态。
- **载入**：从草稿/历史恢复编辑器、选中表与草稿 id。
- **验收**：刷新页面后的持久化行为与后端设计一致（Mock 阶段标注）。

### 6.6 合规与提示（FR-SQL-06）

- 页面显著位置提示：**仅在线查看，不支持导出/分享**（与原型一致）。
- 结果区脚本文案提示截断行数、涉及表数量。

## 7. 权限、安全与审计

- 列表、解析、执行三处权限判断需**同源**，避免仅前端校验。
- 敏感环境应记录：谁、何时、执行了哪段 SQL（摘要）、涉及表、是否拒绝（供审计）。

---

## 8. 验收标准（模块级检查清单）

- [ ] 新建查询、从详情进入、返回路径正确。
- [ ] 左侧仅授权表；多选、聚焦表与字段插入正确。
- [ ] 执行前越权引用被拦截且无数据泄露；合法 SQL 可出结果预览。
- [ ] 草稿保存/更新、历史记录、载入链路正确（或已标注 Mock 限制）。
- [ ] 无权限页与申请权限入口可用。
- [ ] 全站文案与产品一致：**无导出、无分享**。
