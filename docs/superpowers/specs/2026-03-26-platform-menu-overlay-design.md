# 一级菜单浮层设计说明

**目标**

在数据资产管理平台原型的 header 左上角、logo 前新增一个系统级入口按钮。点击后展示一个从 header 下方展开的一级菜单浮层，用于集中展示平台各一级功能模块，并支持从浮层直接跳转到对应页面。

**交互设计**

- 入口按钮采用宫格图标，放置在 header 最左侧。
- 点击按钮后，显示覆盖主内容区域的一级菜单浮层，header 保持可见。
- 浮层左侧展示一级菜单列表。
- 浮层右侧展示当前选中菜单的模块说明、功能点和进入按钮。
- 点击左侧菜单时，右侧说明同步切换。
- 点击右侧“进入模块”按钮时，关闭浮层并跳转到对应页面。
- 点击遮罩、关闭按钮或再次点击入口按钮时，关闭浮层。

**信息架构**

一级菜单与当前原型现有导航保持一致：

- 资产检索
- 资产目录
- 资源目录
- 资源管理
- 血缘追溯
- 资产分析

其中前五个模块映射到现有页面；`资产分析` 暂无落地页面，点击进入时展示“建设中”提示。

**实现方式**

- 在单文件 HTML 中增加一级菜单入口按钮与浮层 DOM。
- 使用一组前端配置数据驱动菜单名称、说明文案与页面映射。
- 复用现有 `switchPage`、`openLineagePage`、`showNotification` 等交互函数，避免破坏当前原型结构。
- 为浮层增加键盘和遮罩关闭能力，保持交互一致性。
