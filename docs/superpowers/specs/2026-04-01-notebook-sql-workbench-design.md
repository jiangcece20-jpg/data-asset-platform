# 即席查询 Notebook 工作台 — 设计文档

> 日期：2026-04-01
> 状态：已确认

## 1. 产品定位

Jupyter 风格的 SQL Notebook 工作台，替代现有的基础 SQL 查询页面。支持多 Cell 渐进式数据探索、Markdown 笔记穿插、查询结果可视化，以及 Notebook 只读分享。

### 核心目标

- 提供"写一段 → 跑一下 → 看结果 → 再写下一段"的渐进式探索体验
- SQL Cell + Markdown Cell + Chart Cell 混合编排，形成可读的数据分析文档
- 支持只读快照分享，方便团队协作

## 2. 整体布局

经典三栏 Notebook 布局（左侧双层面板 + 中间 Notebook 主体）：

```
┌─────────────────────────────────────────────────────────┐
│  Header（深色）  资产检索 / 资产目录 / ... / 即席查询 / 我的  │
├────────────┬────────────────────────────────────────────┤
│            │  工具栏：[标题可编辑] [保存] [分享] [全部运行]   │
│  左侧面板   ├────────────────────────────────────────────┤
│  240px     │                                            │
│            │           Notebook 主体区域                  │
│ ┌────────┐ │                                            │
│ │Notebook│ │   [ SQL Cell ]  ▶ 运行   结果/图表          │
│ │列表     │ │   [ Markdown Cell ]                        │
│ │        │ │   [ SQL Cell ]  ▶ 运行   结果/图表          │
│ │我的(N)  │ │   [ Chart Cell ]                           │
│ │收到(N)  │ │   ...                                      │
│ ├────────┤ │                                            │
│ │表/字段  │ │          + 添加 Cell                        │
│ │辅助     │ │                                            │
│ │ 📋 表A  │ │                                            │
│ │ 📋 表B  │ │                                            │
│ └────────┘ │                                            │
├────────────┴────────────────────────────────────────────┤
│  底部状态栏：连接状态 · 最后保存时间 · Cell数量              │
└─────────────────────────────────────────────────────────┘
```

### 页面入口

一级导航新增"即席查询"菜单项，位于"资源管理"和"我的"之间。

### 左侧面板

- 上半部分：Notebook 列表，分"我的"和"收到的分享"两个分组，支持新建、搜索、删除
- 下半部分：当前权限范围内的数据表列表，点击表名展开字段树，点击字段名插入到当前聚焦的 SQL Cell
- 两部分之间有可拖拽的分割线

### 顶部工具栏

- Notebook 标题（点击可编辑）
- 保存按钮（Ctrl+S）
- 分享按钮 → 生成只读链接
- "全部运行"按钮 → 按顺序执行所有 SQL Cell
- 更多操作（导出、清空输出等）

### 底部状态栏

连接状态 · 最后保存时间 · Cell 数量

## 3. Cell 类型与交互

### 3.1 SQL Cell

- Monaco Editor（CDN），SQL 语法高亮、表名/字段名自动补全
- 编辑器高度自适应内容，最小 3 行，最大 20 行，超出滚动
- 右上角：运行按钮（Ctrl+Enter / Cmd+Enter）、更多菜单（删除、上移/下移、转为 Markdown）
- 执行后结果内联展示在 Cell 下方：执行状态、耗时、行数
- 结果区操作：表格视图（默认）、"转为图表"（在下方插入 Chart Cell）、导出 CSV
- 执行失败显示红色错误信息，支持语法错误行高亮

### 3.2 Markdown Cell

- 默认渲染模式，双击进入编辑模式（textarea）
- 用 marked.js（CDN）渲染
- 点击 Cell 外部或按 Esc 退出编辑，自动渲染

### 3.3 Chart Cell（可视化 Cell）

- 绑定某个 SQL Cell 的查询结果作为数据源
- 图表类型切换：柱状图、折线图、饼图、散点图
- 设置面板：选择 X 轴字段、Y 轴字段、分组字段
- 用 ECharts（CDN）渲染
- 数据源 Cell 重新执行后，图表自动刷新

### 3.4 Cell 通用交互

- Cell 之间有"+ 添加"按钮，hover 时显示，点击选择类型（SQL / Markdown / Chart）
- 拖拽排序：Cell 左侧有拖拽手柄
- 快捷键：Ctrl+Enter 执行当前 SQL Cell，Shift+Enter 执行并跳到下一个 Cell（没有则新建）

## 4. Notebook 管理与分享

### 4.1 生命周期

- 新建：左侧面板"+ 新建 Notebook"，默认标题"未命名 Notebook"并聚焦标题，预置一个空 SQL Cell
- 自动保存：编辑后 3 秒无操作自动保存，状态栏显示保存时间
- 手动保存：Ctrl+S，toast 提示
- 删除：列表右键菜单，二次确认

### 4.2 Notebook 列表

- "我的"分组：按最后修改时间倒序，显示标题、Cell 数量、时间
- "收到的分享"分组：显示标题、分享者、只读标记
- 搜索：按标题模糊匹配
- 当前打开的 Notebook 高亮

### 4.3 分享

- 点击"分享"按钮弹出弹窗
- 生成只读链接，一键复制
- 链接有效期：7天 / 30天 / 永久
- 收到分享的人进入只读模式：Cell 不可编辑，运行按钮隐藏，顶部显示"只读 · 来自 XXX 的分享"横幅
- 提供"复制到我的 Notebook"按钮（Fork 一份到自己空间）

### 4.4 从详情页进入

- 资源详情页保留"去查询"按钮
- 如果用户当前有打开的 Notebook：在末尾追加一个 SQL Cell，预填 `SELECT * FROM 表名 LIMIT 100;`，跳转到该 Cell
- 如果没有打开的 Notebook：创建新的，标题"表名 - 快速查询"，包含预填的 SQL Cell

## 5. 权限与异常处理

### 5.1 权限规则

- 左侧表列表只展示已有查询权限的表
- SQL 执行前权限校验：引用无权限表时拦截，提示具体哪张表无权限，提供"申请权限"链接
- Notebook 增删改仅限本人

### 5.2 Cell 执行状态

| 状态 | 表现 |
|---|---|
| 空闲 | 编辑器正常，无结果区 |
| 执行中 | 运行按钮 loading，左侧蓝色竖条，结果区骨架屏 |
| 成功 | 绿色 ✓ + 行数 + 耗时，展示结果表格 |
| 失败 | 红色 ✗ + 错误信息，语法错误高亮对应行 |

### 5.3 "全部运行"

- 按顺序从上到下执行所有 SQL Cell
- 某 Cell 失败时暂停，提示"Cell #N 执行失败，是否继续？"

### 5.4 异常场景

- 网络断开：状态栏变红"连接断开"，自动保存暂停，恢复后重试
- 查询超时：30 秒超时，提示优化 SQL
- 结果集过大：展示前 1000 行，底部提示截断信息，导出 CSV 不受限

### 5.5 空白态

- 首次进入（无 Notebook）：欢迎页 + "创建你的第一个 Notebook" 按钮
- 空 Notebook：一个空 SQL Cell，placeholder"输入 SQL 开始查询..."

## 6. 技术实现

### 6.1 文件结构

```
work/prototype/notebook.html   ← 独立新文件，不修改任何现有文件
```

### 6.2 外部依赖（CDN）

| 库 | 用途 |
|---|---|
| Monaco Editor | SQL 编辑器 |
| marked.js | Markdown 渲染 |
| ECharts | 图表渲染 |

### 6.3 数据层（Mock）

- `MOCK_NOTEBOOKS[]` — Notebook 列表及 cells 数组
- `MOCK_TABLES[]` — 复用现有表数据结构
- `MOCK_SHARED[]` — 收到的分享列表
- `localStorage` 持久化

### 6.4 样式体系

- 复用 CSS 变量（`--primary: #1677ff` 等 Ant Design 风格）
- 深色 Header + 白色内容区，与 manus.html 视觉统一
- Cell 卡片：白底 + 轻阴影 + 圆角
- 聚焦 Cell 左侧蓝色竖条高亮

### 6.5 核心模块

```
notebook.html <script> 内部模块：
├── Mock 数据
├── 状态管理      — currentNotebook / cells / focusedCellId
├── Notebook 管理  — 新建 / 打开 / 保存 / 删除 / 分享
├── Cell 管理     — 添加 / 删除 / 排序 / 类型切换
├── SQL 执行      — 单 Cell 执行 / 全部运行 / 权限校验
├── 编辑器管理    — Monaco 实例创建 / 销毁 / 自动补全
├── 图表管理      — ECharts 实例 / 数据绑定 / 类型切换
├── Markdown      — marked 渲染 / 编辑切换
├── 面板交互      — 表列表 / 字段树 / 拖拽分割线
└── 快捷键        — Ctrl+Enter / Ctrl+S / Shift+Enter
```
