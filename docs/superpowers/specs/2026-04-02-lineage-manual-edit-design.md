# 血缘关系手动编辑功能 — 设计文档

> 日期：2026-04-02
> 状态：已确认

## 1. 背景

平台的血缘关系目前依赖系统自动采集（SQL 解析、调度依赖等），但存在以下场景需要人工介入：
- 自动采集不准确（误判上下游）
- 自动采集遗漏（跨系统、非 SQL 链路）
- 指标/标签的血缘无法自动解析，需手动配置
- 需要初始化或重置血缘关系

## 2. 资产类型与血缘层级

### 2.1 表级血缘
描述整个资源的上下游关系，支持的资源类型：表、视图、API、报表。

### 2.2 字段级血缘
描述资源内部字段/参数/指标的映射关系，分两种模式：

| 模式 | 适用场景 | 映射方式 |
|------|---------|---------|
| 字段 ↔ 字段 | 表/视图/API/报表 之间 | 多对多：上游资源的多个字段 → 下游资源的多个字段 |
| 字段 → 指标/标签 | 指标或标签 与 表 之间 | 指标/标签整体关联到表的一个或多个字段 |

**字段级映射中各类型的"字段"含义：**
- 表/视图：表字段
- API：请求参数 / 响应参数
- 报表：展示指标
- 指标：指标本身（无内部字段，整体作为一个节点）
- 标签：标签本身（无内部字段，整体作为一个节点）

## 3. 功能入口

### 3.1 入口位置
资源详情页 → 血缘 tab → 右上角工具栏（与"全屏血缘"按钮同行），新增"编辑血缘"按钮。

### 3.2 血缘 tab 展示规则
**重要：血缘 tab 页中的图谱和关系展示的是"有效血缘"，即排除"已排除"状态的关系。用户只有进入"编辑血缘"弹窗后，才能看到哪些关系被排除过。**

## 4. 编辑血缘弹窗

### 4.1 弹窗结构
点击"编辑血缘"后打开大弹窗（宽度 900px），布局：

```
┌─────────────────────────────────────────────────┐
│ 血缘关系管理 — {资源名}                    [×]  │
├─────────────────────────────────────────────────┤
│ [表级血缘]  [字段级血缘]                        │
│                                                 │
│ 筛选器：[搜索资源名] [方向▾] [来源▾] [状态▾]    │
│                                                 │
│ ┌─────────────────────────────────────────────┐ │
│ │ ☐ │方向│资源名│类型│来源方式│状态│操作      │ │
│ │───│────│──────│────│────────│────│──────    │ │
│ │ ☐ │上游│ods.. │ 表 │自动采集│有效│排除      │ │
│ │ ☐ │上游│dim.. │ 表 │手动添加│有效│删除      │ │
│ │ ☐ │下游│ads.. │ 表 │自动采集│已排除│恢复   │ │
│ └─────────────────────────────────────────────┘ │
│                                                 │
│ [+ 添加血缘关系]  [批量删除]  [初始化血缘]      │
└─────────────────────────────────────────────────┘
```

### 4.2 筛选器
- 搜索框：按资源名模糊搜索
- 方向：全部 / 上游 / 下游
- 来源方式：全部 / 自动采集 / 手动添加
- 状态：全部 / 有效 / 已排除（默认"有效"）

### 4.3 字段级血缘 tab
列表列：☐ | 方向 | 上游资源.字段 | → | 下游资源.字段 | 来源方式 | 状态 | 操作

筛选器同上，额外增加：上游资源筛选、下游资源筛选。

## 5. 血缘关系状态

| 状态 | 标签样式 | 来源 | 说明 | 可执行操作 |
|------|---------|------|------|-----------|
| 有效（自动采集） | 灰色标签"自动采集" | 系统 | 系统解析出的关系 | 排除 |
| 有效（手动添加） | 蓝色标签"手动添加" | 用户 | 用户手动创建 | 删除（真删） |
| 已排除 | 红色删除线+灰色文字 | 用户排除 | 用户认为不准确 | 恢复 |

**操作逻辑：**
- **排除**（自动采集的）：不从列表删除，状态变为"已排除"，行样式变灰+删除线。下次系统重新采集时不会恢复。
- **恢复**（已排除的）：状态恢复为"有效（自动采集）"。
- **删除**（手动添加的）：从列表真删，不可恢复。
- 排除和删除操作均需填写原因（可选）。

## 6. 添加血缘关系流程

### 6.1 表级血缘添加
1. 点击"添加血缘关系"
2. 弹出子表单：
   - 方向：上游 / 下游（单选）
   - 目标资源：搜索下拉（支持按名称/中文名搜索，显示类型标签）
   - 修正原因：文本框（可选）
3. 提交后在列表中新增一行，来源方式为"手动添加"

### 6.2 字段级血缘添加
1. 点击"添加字段映射"
2. 弹出子表单，根据当前资源类型自动切换模式：

**模式 A：字段 ↔ 字段（表/视图/API/报表 之间）**
- 选择方向 + 目标资源
- 选好后显示映射区域：每行一条映射
  - 上游字段（下拉） → 下游字段（下拉）
  - 可点击"+ 添加一行"继续添加
  - 可点击"同名自动匹配"：自动匹配上下游同名字段，一键生成多行
- 修正原因（可选）

**模式 B：指标/标签 ↔ 表字段**
- 当前资源是指标或标签时，目标资源只能选表
- 选好表后，显示表的字段列表（多选勾选）
- 勾选的字段即为关联字段
- 修正原因（可选）

### 6.3 批量删除
- 勾选多行 → 点击"批量删除"
- 自动采集的执行"批量排除"，手动添加的执行"批量真删"
- 弹确认框，显示将排除 N 条 + 删除 M 条

## 7. 初始化血缘

- 点击"初始化血缘"按钮
- 弹出确认框：「将触发系统重新采集当前资源的血缘关系，现有的手动添加关系将保留，自动采集关系将被刷新。确认执行？」
- 确认后：
  - 清除所有"自动采集"和"已排除"状态的关系
  - 保留"手动添加"的关系
  - 触发系统重新解析（mock 中模拟为重新生成自动采集数据）
  - 显示"正在采集中…"loading 状态，完成后刷新列表

## 8. 操作记录

所有血缘编辑操作记录到资源的操作记录中：
- 操作类型：血缘新增 / 血缘排除 / 血缘恢复 / 血缘删除 / 血缘初始化
- 记录内容：操作人、时间、具体变更（如"排除上游关系：ods_order_raw → dwd_order_detail"）、原因

## 9. 权限控制

- 只有资源的技术负责人和业务负责人可以编辑血缘
- 超管可以编辑所有资源的血缘
- 非负责人只能查看，"编辑血缘"按钮置灰并 hover 提示"仅负责人可编辑"
