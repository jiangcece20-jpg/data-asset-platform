# AI 找数功能 — 设计文档 (Design Spec)

## 1. 文档信息

- 文档版本：V1.0
- 文档日期：2026-04-01
- 文档状态：设计稿
- 适用范围：数据资产管理平台 — AI 找数功能

---

## 2. 背景与问题

当前数据资产管理平台的"资源发现"依赖关键词搜索 + 筛选条件，用户需要知道准确的表名、英文名或分类才能找到数据。存在以下问题：

- 用户描述的是业务语言（"我想看用户下单后的退款率"），但数据资产用的是技术语言（`dwd_order_refund_detail`）
- 不同角色关心的资源类型不同 — 分析师找报表，开发找表和 API，运营找看板
- 现有搜索是精确匹配思路，缺乏语义理解和推荐能力
- 用户找到一个资源后，不知道还有哪些相关的上下游资源可以配合使用
- 平台已有数千个资源，用户靠浏览和筛选效率低下

---

## 3. 产品目标

### 3.1 业务目标

- 降低用户找数门槛，让不懂技术命名的业务用户也能快速定位数据
- 通过角色感知推荐，提升不同角色的找数效率
- 提升平台资源的曝光率和使用率，尤其是已有但用户不知道的报表和看板

### 3.2 用户目标

- 用自然语言描述需求，就能找到相关的表、报表、API、看板等资源
- 不需要知道技术表名，AI 能理解业务意图
- 推荐结果符合自己角色的使用习惯，不需要在大量无关资源中翻找

### 3.3 设计目标

- 提供明显区别于常规搜索的交互体验
- 对话式交互，支持多轮追问和意图缩小
- 角色推荐策略对用户透明，无需手动选择

---

## 4. 功能边界

### 4.1 本期范围

- AI 找数入口（搜索框旁按钮 + 中央对话弹窗）
- 自然语言意图理解与资源推荐
- 基于角色的推荐策略（六种细分角色）
- 多轮对话与追问能力
- 推荐结果卡片 + 弹窗内摘要展开
- 对话历史持久保留
- 追问引导标签
- 原型阶段使用 mock 数据

### 4.2 暂不纳入本期

- 指标值查询（"上个月 GMV 是多少"）
- 真实 LLM + 向量检索后端
- 对话历史的搜索和管理
- 资源推荐的反馈机制（点赞/点踩）
- 跨资源的关联推荐（血缘上下游自动推荐）

## 5. 用户角色与推荐策略

### 5.1 角色推断机制

系统基于以下字段自动推断用户的细分角色，用户无感：

- 部门
- 职位 / 岗位
- 所属业务线
- 平台行为数据（埋点统计）

### 5.2 细分角色与推荐偏好

| 细分角色 | 推断依据（示例） | 推荐资源优先级 |
|---|---|---|
| 数据分析师 | 数据部 + 分析岗 | 报表 > 宽表 > 指标口径 |
| 数仓/数据开发 | 数据部/技术部 + 数据开发岗 | 表 > 视图 > 血缘上下游 |
| 后端开发 | 技术部 + 开发岗（非数据） | API > 表 |
| 业务运营 | 运营部 / 市场部 | 报表 > 看板 |
| 产品经理 | 产品部 | 报表 > 指标口径 > 看板 |
| 管理层 | 总监及以上职级 | 报表 > 看板 |

### 5.3 推断优先级

1. 优先使用部门 + 职位/岗位进行规则匹配
2. 规则无法匹配时，降级为平台行为数据推断（常搜资源类型、常访问页面）
3. 行为数据不足时，使用默认策略（报表 > 表 > API > 看板 > 指标口径 > 视图 > 标签）

## 6. 资源范围

AI 找数覆盖以下资源类型：

| 资源类型 | 说明 |
|---|---|
| 表 | 数仓各层表（ODS/DWD/DWS/ADS） |
| 视图 | 数据库视图 |
| API | 数据服务接口 |
| 报表 | BI 报表 |
| 看板 | Dashboard 看板 |
| 指标（口径） | 指标定义、计算逻辑、数据来源（不含指标值） |
| 标签 | 数据标签 |

## 7. 交互设计

### 7.1 入口

- 位置：现有搜索框右侧，增加"AI 找数"按钮
- 样式：紫色渐变背景，带 AI 图标，与常规搜索形成明显视觉区分
- 点击后弹出中央对话弹窗

### 7.2 对话弹窗

- 类型：居中模态弹窗，带半透明遮罩
- 尺寸：宽 720px，最大高度 80vh
- 圆角：16px
- 动画：从下方滑入 + 淡入

#### 弹窗结构（从上到下）

1. **头部**
   - 左侧：AI 图标 + "AI 找数助手" 标题 + 副标题"用自然语言描述你想找的数据，我来帮你定位"
   - 右侧：关闭按钮

2. **对话区域**（可滚动）
   - AI 欢迎语
   - 用户消息气泡（右侧，蓝色）
   - AI 回复气泡（左侧，浅灰背景）+ 资源推荐卡片 + 追问引导标签

3. **输入区域**
   - 输入框 + 发送按钮
   - placeholder："描述你想找的数据，例如：各城市的配送时效报表"

#### 关闭方式

- 点击右上角关闭按钮
- 点击遮罩区域
- 按 Esc 键

### 7.3 AI 回复结构

每次 AI 回复包含以下部分：

1. **意图解释**（文字气泡）
   - 示例："我理解你想查看用户下单后的退款相关数据。根据你的角色，我优先为你推荐了报表和明细表："

2. **推荐资源卡片**（1-5 张）
   - 每张卡片展示：
     - 资源类型标签（彩色标签：表=蓝、报表=绿、API=橙、看板=紫）
     - 资源名称（英文名 + 中文名）
     - 来源系统
     - AI 匹配原因（一句话说明为什么推荐这个资源）

3. **追问引导标签**（0-4 个）
   - 圆角胶囊标签，点击后作为用户输入发送
   - 示例："按渠道看退款"、"退款原因分布"、"关联的订单表"

### 7.4 AI 追问策略

采用置信度判断机制，根据用户输入的信息量决定是直接给结果还是追问。

#### 置信度判断维度

AI 从三个维度评估用户输入的置信度：

**维度一：业务域是否明确**
- ✅ 明确："退款相关的数据" → 业务域 = 交易/退款
- ✅ 明确："骑手配送时效" → 业务域 = 供应链/配送
- ❌ 模糊："用户相关的数据" → 用户域太大（行为？画像？交易？留存？）

**维度二：资源粒度是否可推断**
- ✅ 可推断："有没有退款的日报" → 明确要报表
- ✅ 可推断："dwd_order 开头的表" → 明确要表
- ❌ 不可推断："退款的数据" → 要报表？明细表？API？（角色偏好可部分兜底）

**维度三：匹配结果的区分度**
- ✅ 区分度高：候选资源 3-5 个，且 top1 相关度远高于其他
- ❌ 区分度低：候选资源 20+，相关度分数都差不多

#### 置信度策略规则（产品侧定义）

| 置信度 | 触发条件 | AI 行为 | 示例 |
|---|---|---|---|
| 高 | 业务域明确 + 资源类型可推断 + 候选结果区分度高 | 直接返回推荐结果，不追问 | "dwd_order_refund_detail 这张表" → 直接返回 |
| 中 | 业务域明确 + 资源类型模糊，但角色偏好能排出合理优先级 | 先给结果，底部附追问引导标签 | "退款相关的数据" → 给退款域的报表和表，附"按渠道看退款"等引导 |
| 低 — 业务域模糊 | 用户输入涉及的业务域太宽泛 | 主动追问缩小业务域 | "用户相关的数据" → "你想看的是用户行为、用户画像、还是用户交易？" |
| 低 — 输入太短 | 用户输入仅 1-2 个词，信息量不足 | 主动追问补充上下文 | "订单" → "你想看订单的哪个方面？退款、履约、还是支付？" |
| 低 — 结果太多 | 候选资源数量多且相关度分数接近 | 主动追问缩小范围 | "交易数据" → "你需要的是交易明细、交易汇总、还是交易异常监控？" |

#### 追问体验约束

- **最多连续追问 2 次**：超过 2 次仍无法收敛时，强制按当前最佳理解给出结果，避免用户觉得"找个数还要聊半天"
- **追问必须给选项**：追问时必须提供 2-4 个可点击的选项，不要让用户自己想怎么回答
- **中置信度不阻塞**：中置信度时先给结果，追问引导标签只是辅助，用户可以忽略

#### 角色分工说明

- **产品负责**：定义上述策略规则、追问场景、话术模板、体验约束（即本节内容）
- **算法负责**：将策略规则翻译为可量化的技术实现（意图分类器置信度阈值、向量检索相关度分数阈值、候选数量阈值等）
- **上线后共同迭代**：产品和算法一起看实际对话日志，持续调优阈值和策略

### 7.5 卡片展开摘要

用户点击推荐卡片后，卡片在弹窗内就地展开，显示更多信息：

- 资源描述（完整描述文本）
- 负责人
- 热度（近30天查询量/浏览量）
- 权限状态（已有权限 / 申请权限 / 申请中）
- 字段列表预览（前 5 个字段）
- "查看详情"按钮 → 关闭弹窗，跳转到统一详情页

再次点击卡片可收起摘要。

### 7.6 对话历史

- 对话历史持久化保存（localStorage 或后端存储）
- 用户关闭弹窗后再次打开，上次对话仍在
- 弹窗头部提供"新对话"按钮，用户可主动开启新会话
- 历史对话不设上限，超出弹窗高度时可滚动查看

## 8. 状态说明

### 8.1 AI 找数按钮

- 默认态：紫色渐变背景
- Hover 态：轻微放大 + 透明度变化
- 激活态（弹窗打开时）：保持高亮

### 8.2 推荐卡片

- 默认态：浅灰边框
- Hover 态：蓝色边框 + 浅蓝阴影
- 展开态：卡片高度增加，显示摘要信息
- 点击"查看详情"后：关闭弹窗，跳转详情页

### 8.3 追问引导标签

- 默认态：灰色边框 + 灰色文字
- Hover 态：蓝色边框 + 蓝色文字 + 浅蓝背景

### 8.4 AI 加载态

- AI 正在思考时，显示打字动画（三个跳动的点）
- 输入框和发送按钮在 AI 回复完成前禁用

## 9. 后端架构方案（本期原型不实现，供后续参考）

### 9.1 整体架构：LLM + 向量检索混合

```
用户输入
  ↓
意图理解（LLM）→ 提取关键词、业务域、资源类型偏好
  ↓
向量检索 → 从全量资源中召回 Top 20-50 候选
  ↓
精排（LLM）→ 结合用户角色偏好重排序，生成匹配原因
  ↓
返回 Top 3-5 推荐结果
```

### 9.2 数据准备清单

后端和数据团队需要准备以下内容：

#### 资源元数据索引

需要为每个资源构建可检索的元数据文档，包含：

| 字段 | 说明 | 用途 |
|---|---|---|
| 资源 ID | 唯一标识 | 关联详情 |
| 英文名 | 技术名称 | 精确匹配 |
| 中文名 | 业务名称 | 语义匹配 |
| 资源类型 | 表/视图/API/报表/看板/指标/标签 | 角色过滤 |
| 描述 | 资源描述文本 | 语义匹配 |
| 标签 | 资源标签列表 | 语义匹配 |
| 字段列表 | 字段名 + 字段注释 | 语义匹配 |
| 所属业务域 | 业务域归属 | 意图匹配 |
| 来源系统 | 数据来源 | 结果展示 |
| 负责人 | 资源负责人 | 结果展示 |
| 热度数据 | 近30天查询量/浏览量 | 排序因子 |
| 权限信息 | 当前用户的权限状态 | 结果展示 |

#### 向量索引

- 将每个资源的"中文名 + 描述 + 标签 + 字段注释"拼接为文本，生成 embedding 向量
- 建议使用的 embedding 模型：text-embedding-ada-002 或同等中文语义模型
- 向量数据库：Milvus / Qdrant / Weaviate 均可
- 索引需支持增量更新（资源新增/修改时自动重建向量）

#### 用户画像数据

- 用户部门、职位/岗位、所属业务线（从用户中心/HR 系统同步）
- 用户平台行为统计（从埋点系统聚合）：
  - 近30天搜索的资源类型分布
  - 近30天访问的资源类型分布
  - 近30天收藏的资源列表

#### LLM 接入

- 需要接入一个支持中文的 LLM API（GPT-4 / 文心一言 / 通义千问等）
- 用途：意图理解、追问判断、匹配原因生成、结果精排
- 预估单次对话 token 消耗：输入 ~2000 tokens，输出 ~500 tokens

#### 角色推断服务

- 基于部门 + 职位的规则引擎（优先）
- 基于行为数据的统计推断（降级）
- 输出：用户的细分角色标签 + 对应的资源类型优先级权重

## 10. 原型阶段 Mock 策略

原型阶段不接入真实后端，使用以下 mock 策略：

### 10.1 Mock 资源库

预置 30-50 条覆盖各类型的 mock 资源数据，涵盖：
- 交易域（订单、退款、支付）
- 用户域（画像、行为、留存）
- 供应链域（库存、配送）
- 营销域（活动、优惠券）

### 10.2 Mock 意图匹配

基于关键词匹配模拟 AI 行为：
- 用户输入包含"退款" → 返回退款相关资源
- 用户输入包含"用户" → 根据角色返回不同类型的用户相关资源
- 无法匹配时 → 触发追问

### 10.3 Mock 角色切换

原型中提供一个隐藏的角色切换入口（如弹窗底部小字），方便演示不同角色看到不同推荐结果的效果。

## 11. 用户故事

### Story 1：自然语言找数

作为数据分析师，我想用自然语言描述我的数据需求，以便快速找到相关的报表和数据表，而不需要知道技术表名。

验收标准：
- 输入"用户下单后的退款情况"，能返回退款相关的报表和表
- 推荐结果中报表排在表前面（符合分析师角色偏好）
- 每个结果有 AI 匹配原因说明

### Story 2：角色感知推荐

作为后端开发，我搜索同样的关键词时，看到的推荐结果优先级与分析师不同。

验收标准：
- 同样搜索"用户画像"，后端开发优先看到 API，分析师优先看到报表
- 角色识别自动完成，用户无需手动选择

### Story 3：多轮追问

作为业务运营，我输入了一个模糊的需求，AI 能追问帮我缩小范围。

验收标准：
- 输入"用户相关的数据"，AI 追问"你想看的是用户行为、用户画像、还是用户交易？"
- 用户选择后，AI 返回对应范围的资源

### Story 4：卡片展开摘要

作为用户，我想在不离开对话的情况下快速判断一个资源是否是我要的。

验收标准：
- 点击推荐卡片，在弹窗内展开摘要（描述、负责人、字段预览等）
- 确认后点击"查看详情"跳转到详情页
- 再次点击卡片可收起摘要

### Story 5：对话历史保留

作为用户，我关闭对话弹窗去看了详情页，回来后想继续之前的对话。

验收标准：
- 关闭弹窗后再次打开，对话历史仍在
- 可以继续追问
- 可以点击"新对话"开启全新会话
